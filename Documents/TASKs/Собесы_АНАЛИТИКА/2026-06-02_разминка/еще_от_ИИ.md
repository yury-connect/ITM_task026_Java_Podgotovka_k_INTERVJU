## Блок1: `String Pool` и интернирование

```Java
// Пример 1
String s1 = "Java";
String s2 = "Ja" + "va";
System.out.println(s1 == s2); // 1.1) 
System.out.println(s1.equals(s2)); // 1.2) 

// Пример 2
String s3 = "Ja";
String s4 = s3 + "va";
System.out.println(s1 == s4); // 1.3) 
System.out.println(s1.equals(s4)); // 1.4) 

// Пример 3
final String s5 = "Ja";
String s6 = s5 + "va";
System.out.println(s1 == s6); // 1.5) 

// Пример 4
String s7 = new String("Java").intern();
System.out.println(s1 == s7); // 1.6) 
```


### Блок 2: Кэширование целых чисел (Integer Cache)
```Java
// Пример 5
Integer a1 = 127;
Integer a2 = 127;
System.out.println(a1 == a2); // 2.1) 
System.out.println(a1.equals(a2)); // 2.2) 

// Пример 6
Integer b1 = -128;
Integer b2 = -128;
System.out.println(b1 == b2); // 2.3) 

// Пример 7
Integer c1 = Integer.valueOf(100);
Integer c2 = Integer.valueOf(100);
System.out.println(c1 == c2); // 2.4) 
```

## 🔑 Ответы и разбор «почему так?»

Не подглядывай, пока не решишь всё сам! 🙂

- **1.1) `true`**: Обе строки — это литералы. Компилятор оптимизирует выражение `"Ja" + "va"` еще на этапе компиляции, превращая его в `"Java"`, и берет одну ссылку из String Pool.
    
- **1.2) `true`**: Метод `equals()` у `String` сравнивает посимвольное содержимое строк. Оно идентично.
    
- **1.3) `false`**: Так как переменная `s3` не является константой, сложение строк `s3 + "va"` происходит в рантайме через `StringBuilder`. Создается **новый** объект в куче (Heap) вне пула строк. Ссылки разные.
    
- **1.4) `true`**: Символы внутри строк одинаковые (`"Java"`).
    
- **1.5) `true`**: Ловушка! Переменная `s5` помечена как `final`. Компилятор знает, что она никогда не изменится, и трактует её как константу времени компиляции (Compile-time constant). Выражение `s5 + "va"` оптимизируется до `"Java"` на этапе сборки и ссылается на String Pool.
    
- **1.6) `true`**: Метод `.intern()` принудительно ищет строку в String Pool. Если она там есть, он возвращает ссылку из пула, а не на объект в куче, созданный через `new`.
    

- **2.1) `true`**: В Java для класса `Integer` работает встроенный кэш для значений в диапазоне **от -128 до 127** включительно. При автоупаковке числа `127` Java возвращает ссылки на один и тот же заранее созданный объект в кэше.
    
- **2.2) `true`**: `equals()` у оберток сравнивает их внутренние примитивные значения. Они равны.
    
- **2.3) `true`**: `-128` — это нижняя граница Integer Cache, ссылки берутся из кэша.
    
- **2.4) `true`**: Метод `Integer.valueOf()` под капотом использует автоупаковку и честно обращается к Integer Cache. Поскольку 100 входит в диапазон $[-128; 127]$, объекты идентичны.
    

- **3.1) `Нет`**: Эта строка **не скомпилируется**! Оператор `==` между двумя разными классами-обертками (`Integer` и `Long`) запрещен, так как они находятся в разных ветках наследования и компилятор знает, что ссылки никогда не совпадут.
    
- **3.2) `false`**: Метод `equals()` у любого класса-обертки (`Long`, `Integer`, `Double`) первым делом проверяет тип через `instanceof`. Если типы не совпадают (мы пытаемся сравнить `Long` с `Integer`), метод сразу возвращает `false`, даже не сравнивая числа внутри.
    
- **3.3) `true`**: Жесткая ловушка! Здесь сравниваются обертка `Long` и примитив `int`. Когда один из участников оператора `==` является **примитивом**, Java автоматически распаковывает обертку (`Long` превращается в примитивный `long`). После этого `long` и `int` сравниваются как обычные числа. Число 5 равно числу 5.
    
- **3.4) `false`**: Метод `equals(Object o)` принимает объект. Примитив `primitiveInt` (5) автоматически упаковывается в объект `Integer`. Далее срабатывает правило из пункта 3.2 — типы `Long` и `Integer` не равны.
    
- **3.5) `true`**: Примитивные `0.0` и `-0.0` при сравнении через `==` считаются равными.
    
- **3.6) `false`**: Высший пилотаж Java! Метод `equals()` у класса `Double` сравнивает биты чисел через `Double.doubleToLongBits()`. На уровне битового представления в стандарте IEEE 754 знак минус меняет один бит, поэтому биты у `0.0` и `-0.0` разные, и `equals()` возвращает `false`.
