# Что вернёт метод?
```java
static int f() {
    try {
        return 1;
    } finally {
        return 2;
    }
}
```

### **Ответ:**
Метод вернёт **`2`**.
> **`1`** просто пере затрётся и потеряется.

---
### **Почему?**
В Java блок `finally` выполняется **всегда** после завершения блока `try` (кроме случаев аварийного завершения JVM).  
Если в `try` есть `return`, то сначала вычисляется возвращаемое значение (`1`), но перед фактическим возвратом управления вызывающему коду выполняется блок `finally`.

Если в `finally` тоже есть `return`, то **этот `return` перезаписывает результат** из `try`.

#### **Как это работает по шагам:**
1. Вход в `try`.    
2. Встречается `return 1` → вычисляется значение `1` и сохраняется как предварительный результат.    
3. Выполняется блок `finally`.    
4. В `finally` встречается `return 2` → **этот результат подменяет предыдущий**.    
5. Метод возвращает `2`.    

---
### **Дополнительно: если бы не было `return` в `finally`**

Если бы `finally` не содержал `return`, метод вернул бы `1`:
```java
static int f() {
    try {
        return 1;
    } finally {
        // что-то делаем, но не возвращаем
    }
}

// вернёт 1
```

---
### **Предупреждение:**

Такой код считается **плохой практикой**, потому что:
- `return` в `finally` может "проглотить" исключения из `try`.    
- Затрудняет чтение и понимание кода.    

Никогда не пишите `return` в `finally`, если только нет очень веской причины (которой обычно нет).
