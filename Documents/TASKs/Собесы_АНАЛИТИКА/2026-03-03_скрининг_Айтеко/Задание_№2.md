# Задание_№2

```java
public static void main(String[] args) {
    List<Integer> list = new ArrayList<>();
    list.add(1);
    list.add(2);
    list.add(3);
				
    for (Integer integer : list) {
        list.remove(1);
    }
}
```

---
## Этот код выбросит **`ConcurrentModificationException`**.

### Почему?
В Java коллекции (например, `ArrayList`) имеют внутренний счетчик модификаций. Когда ты создаешь `for-each` цикл, он неявно создает **Iterator**.

**Последовательность действий:**
1. Итератор запоминает состояние коллекции    
2. Внутри цикла ты напрямую вызываешь `list.remove(1)`    
3. Коллекция изменяется, но итератор об этом не знает    
4. При следующей итерации итератор проверяет - "эй, структура изменилась!"    
5. Получаешь **ConcurrentModificationException**    

### Как исправить?

**Вариант 1 - Удалять через итератор:**
```java
Iterator<Integer> iterator = list.iterator();
while (iterator.hasNext()) {
    iterator.next(); // обязательно вызвать перед remove
    iterator.remove(); // удаляем текущий элемент
}
```

**Вариант 2 - Копия коллекции:**
```java
for (Integer integer : new ArrayList<>(list)) {
    list.remove(1);
}
```

**Вариант 3 - Обычный for с индексами (иди с конца):**
```java
for (int i = list.size() - 1; i >= 0; i--) {
    list.remove(i);
}
```

### Интересный момент:
Если бы ты удалял **последний элемент**:
```java
for (Integer integer : list) {
    list.remove(2); // удаляем индекс 2 (последний)
}
```

Код бы **НЕ упал**! Потому что итератор проверяет наличие следующего элемента только перед следующей итерацией, а тут итерации закончились. Но так делать всё равно не стоит 😊
