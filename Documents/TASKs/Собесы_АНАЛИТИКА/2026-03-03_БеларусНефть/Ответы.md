# Задание. "ЧТО НЕ ТАК" - Проревьювить код.

```java
@Component  
public class InvoiceProcessingService {  
  
    @Autowired  
    private final InvoiceRepository invoiceRepository;  
    @Autowired  
    private final PricingService pricingService;  
    @Autowired  
    private final TransactionTemplate transactionTemplate;  
    @Autowired  
    private final KafkaTemplate<String, String> kafkaTemplate;  
  
    public void generateInvoice(Long clientId, List<InvoiceItem> items) {  
        BigDecimal total = pricingService.calculateTotal(items); //вызов внешнего сервиса  
  
        Invoice invoice = new Invoice(clientId, total);  
        invoiceRepository.save(invoice);  
  
        String event = String.format(  
                "{\"invoiceId\":%d,\"clientId\":%d,\"total\":\"%s\"}",  
                invoice.getId(), clientId, total  
        );  
  
        kafkaTemplate.send("invoice-events", clientId.toString(), event);  
    }  
}  
  
class InvoiceItem {  
    private Long id;  
    private String productId;  
    private int quantity;  
  
    public InvoiceItem() {  
    }  
  
    public InvoiceItem(Long id, String productId, int quantity) {  
        this.id = id;  
        this.productId = productId;  
        this.quantity = quantity;  
    }  
  
    public Long getId() {  
        return id;  
    }  
  
    public void setId(Long id) {  
        this.id = id;  
    }  
  
    public String getProductId() {  
        return productId;  
    }  
  
    public void setProductId(String productId) {  
        this.productId = productId;  
    }  
  
    public int getQuantity() {  
        return quantity;  
    }  
  
    public void setQuantity(int quantity) {  
        this.quantity = quantity;  
    }  
}  
  
class Invoice {  
    private Long id;  
    private Long clientId;  
    private BigDecimal total;  
  
    public Invoice(Long clientId, BigDecimal total) {  
        this.clientId = clientId;  
        this.total = total;  
    }  
  
    public Long getId() {  
        return id;  
    }  
  
    public Long getClientId() {  
        return clientId;  
    }  
  
    public BigDecimal getTotal() {  
        return total;  
    }  
}
```

# **Code Review** (уровень middle+)
## 🚨 1. ❌ Неправильный **DI**
```java
@Autowired  
private final InvoiceRepository invoiceRepository;
```
🔴 Нельзя `@Autowired` + `final` <u>без конструктора</u>.  
*Spring* не сможет проинициализировать.

✅ Нужно **constructor injection**:
```java
private final InvoiceRepository invoiceRepository;  
  
public InvoiceProcessingService(  
    InvoiceRepository invoiceRepository,  
    PricingService pricingService,  
    TransactionTemplate transactionTemplate,  
    KafkaTemplate<String, String> kafkaTemplate  
) {  
    this.invoiceRepository = invoiceRepository;  
    ...  
}
```
> 📌 Best practice.
- Либо `@RequiredArgsConstructor` над классом

---
## 🚨 2. ❌ Нет транзакции

`invoiceRepository.save(invoice);`  
#### Но нет:
- `@Transactional`    
- или использования `transactionTemplate`    
⚠ Если Kafka отправится, а БД откатится → рассинхрон.
#### 📌 Нужно:
- либо `@Transactional`    
- либо **Outbox pattern** (лучше)    

---
## 🚨 3. ❌ Нет гарантии согласованности **`БД` + `Kafka`**
#### Сейчас:
1. Сохраняем в БД    
2. Отправляем в Kafka    
Если Kafka упадёт → инвойс в БД есть, события нет.
📌 Это классическая проблема distributed transaction.
#### ✅ Решение:
- **Outbox pattern**    
- Kafka transactional producer    

---
## 🚨 4. ❌ Вызов внешнего сервиса вне контроля

pricingService.calculateTotal(items);

⚠ Нет:
- timeout    
- retry    
- circuit breaker    

📌 В проде это риск.

---

## 🚨 5. ❌ Ручная сборка JSON

String.format("{...}")

⚠ Хрупко:
- ошибка формата    
- проблемы с escaping    
- нет версионирования    

✅ Использовать ObjectMapper.

---

## 🚨 6. ❌ Entity не является JPA-сущностью

`Invoice`:
- Нет `@Entity`    
- Нет `@Id`    
- Нет `@GeneratedValue`    
- Нет пустого конструктора    

📌 Такой класс JPA не сохранит.

---

## 🚨 7. ❌ Отсутствует обработка ошибок Kafka

`kafkaTemplate.send(...)`  
Возвращает `ListenableFuture`, но результат игнорируется.

⚠ Нет обработки send failure.

---

## 🚨 8. ❌ Нет валидации входных данных

- `clientId == null`    
- `items == null`    
- пустой список    
- quantity <= 0    

📌 Сервис уязвим.

---

## 🚨 9. ❌ Нет идемпотентности

Если метод вызовется дважды → будет 2 инвойса.

В распределённой системе это проблема.

---

## 🚨 10. ❌ Нарушение SRP

Сервис:
- считает цену    
- сохраняет    
- формирует JSON    
- публикует в Kafka    

📌 Слишком много ответственности.

---

# 🧠 Как ответить на собеседовании

> Основные проблемы:  
> – неправильный DI  
> – нет транзакционности  
> – нет гарантии консистентности БД и Kafka  
> – ручная сериализация JSON  
> – нет обработки ошибок и валидации  
> – нет идемпотентности  
> – сущность не оформлена как JPA entity

Этого достаточно, чтобы показать зрелость уровня middle+.
