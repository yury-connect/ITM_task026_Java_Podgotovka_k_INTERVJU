## **Payment Service**
### Задачи:
- Рассчитывает и формирует стоимость доставки товара;
- Выставляет счет на оплату;
- Контролирует оплату пользователя за доставку: Принимает от банка оплаченный счет и уведомляет об этом **Delivery Service** через соответствующий `kafka-topic: paymant.payment-status` статусом `bill_completed` или `bill_canceled` в случае аннулирования счета;
- Актуализирует из внешних источников инфу для расчета стоимости доставки;
### Стек:
- DB: Postgres;

---
### **Платежи** (Paymants):

| Метод | Эндпоинт                     | Описание (кратко)                           | Бизнес-правила                                                                                                  |
| ----- | ---------------------------- | ------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| GET   | `/api/v1/payments/{id}/cost` | Рассчитать стоимость *(просто узнать цену)* | с ролью SERVICE Базовая_ставка + вес*10 + расстояние*5                                                          |
| POST  | `/api/v1/payments/bill`      | Сформировать и предоставить счет на оплату  | синхронно идет за расчетами в `Payment Service`  с ролью `SERVICE`                                              |
| POST  | `/api/v1/payments/callback`  | оплаченный счет, возвращает сам банк        | Если оплата клиентом прошла успешно, то банк высылает на указанный эндпоинт оплаченный счет/ Только роль `BANK` |

### Формирование счета:
1. Исходя из `Базоваой_ставки`, `веса` груза и `расстояния` формируется стоимость доставки `cost`;
2. Вычисленная величина `cost` включается в сущность `bill` и записывается в БД. Счету присваивается уникальный `id` с типом `UUID`;
3. Клиенту возвращается счет для оплаты `bill`, который включает `сумму` для оплаты,  `id` платежа и срок действия счета `durationDays` в календарных днях;
4. Далее `Delivery Service` бронирует одно из имеющихся в автопарке транспортных средств на указанное в счете время. По истечении указанного интервала `durationDays` счет будет аннулирован, либо оплачен;
5. Если из топика Kafka вычитано событие, что счет аннулирован `bill_canceled` - то ТС разбронируются в свободные. Если вычитано событие `bill_completed` - то начинается процесс доставки товара.

#### Сущность `bill` - эта сущность хранится в БД

| имя переменной | тип                            | Пояснение                                                                       |
| -------------- | ------------------------------ | ------------------------------------------------------------------------------- |
| `id`           | `UUID`                         | Уникальный идентификатор счета                                                  |
| `createdAt`    | `Instant` / `TIMESTAMPTZ`      | Дата (часы/ минуты) **выставления** счета <br>(тип в коде/ тип в БД)            |
| `description`  | `String` / `VARCHAR(500)`      | Описание услуги/ товара                                                         |
| `clientId`     | `Long` / `Long`                | `id` залогиненного пользователя, кот. формирует счет.                           |
| `amount`       | `BigDecimal` / `NUMERIC(19,2)` | Стоимость в рублях <br>(2 знака после запятой)                                  |
| `dueDate`      | `Instant` / `TIMESTAMPTZ`      | Срок оплаты                                                                     |
| `paidAt`       | `Instant` / `TIMESTAMPTZ`      | Дата (часы/ минуты) <br>когда счет был фактически **оплачен**                   |
| `status`       | `Enum BillStatus` / `Enum`     | Состояние счета на текущий момент (enum: `ACTIVE`, `PAID`, `VOIDED`, `EXPIRED`) |
| `voidedAt`     | `Instant` / `TIMESTAMPTZ`      | дата фактического аннулирования                                                 |

```java
// 🔥 Исправленная сущность
@Entity
@Table(name = "payment_bill")
public class Bill {
    private UUID id;   // Уникальный идентификатор счета
    private Instant createdAt;   // Дата выставления счета
    private String description;   // Описание услуги/ товара
    private Long clientId;   // id залогиненного пользователя
    private BigDecimal amount;   // Стоимость в рублях (2 знака после запятой)
    private LocalDate dueDate;  // Срок оплаты
    private BillStatus status;    // Состояние счета на текущий момент
    private Instant paidAt;   // Дата фактической оплаты
    private Instant voidedAt;   // Дата аннулирования
}
```

```sql
CREATE TABLE payment_bill (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    created_at TIMESTAMPTZ NOT NULL,
    description VARCHAR(500),
    client_id BIGINT NOT NULL,
    amount NUMERIC(19, 2) NOT NULL,
    due_date DATE NOT NULL,      -- Срок оплаты (день, до которого нужно оплатить)
    status VARCHAR(30) NOT NULL DEFAULT 'ACTIVE',
    paid_at TIMESTAMPTZ,
    voided_at TIMESTAMPTZ,       -- Дата аннулирования
    
    CONSTRAINT valid_paid CHECK (
        (status = 'PAID' AND paid_at IS NOT NULL) OR
        (status != 'PAID' AND paid_at IS NULL)
    ),
    CONSTRAINT valid_voided CHECK (
        (status = 'VOIDED' AND voided_at IS NOT NULL) OR
        (status != 'VOIDED' AND voided_at IS NULL)
    )
);
```

#### Сущность `BillStatus` - это статус счета
```java
public enum BillStatus {
    ACTIVE,     // Счет активен (ожидает оплаты)
    PAID,       // Оплачен 
    VOIDED,     // Аннулирован (вручную) 
    EXPIRED     // Срок оплаты истек 
}
```

#### DTO `bill` - эта сущность передается клиенту для оплаты, <br>а так-же банк будет возвращать оплаченный счет обратно на платформу

| имя переменной | тип                            | Пояснение                                                         |
| -------------- | ------------------------------ | ----------------------------------------------------------------- |
| `id`           | `UUID`                         | Уникальный идентификатор счета                                    |
| `createdAt     | `Instant` / `TIMESTAMPTZ`      | Дата (часы/ минуты) **выставления** счета <br>(тип в коде и в БД) |
| `description`  | `String` / `VARCHAR(500)`      | Описание услуги/ товара                                           |

| `amount`       | `BigDecimal` / `NUMERIC(19,2)` | Стоимость в рублях <br>(2 знака после запятой)                    |
| `paidAt`       | `Instant` / `TIMESTAMPTZ`      | Дата (часы/ минуты) <br>когда счет был фактически **оплачен**     |

---
Банковская система залогинивается под ролью BANK и отправляет на этот эндпоинт оплаченный счет, что свидетельствует об оплате данного счета. Все поля счета должны совпасть. После этого счет считается оплаченным и берется в работу, а в ответ бан получает статус 200, что свидетельствует о принятии платформой сообщения банка об оплате.. 

## 🔄 Сценарий работы 
```text
1. Delivery Service → POST /payments/cost (рассчет стоимости)
   ↓
2. Delivery Service → POST /payments/bill (создание счета)
   ↓
3. Payment Service:
   - Считает cost = baseRate + weight*10 + distance*5
   - Сохраняет Bill в БД со статусом ACTIVE
   - Устанавливает dueDate = now + durationDays
   ↓
1. Payment Service → возвращает BillResponse клиенту на оплату
   ↓
2. Клиент оплачивает → Банк отправляет POST /payments/callback (оплаченный счет)
   ↓
3. Payment Service:
   - Находит Bill по id
   - Проверяет статус (если ACTIVE → PAID)
   - Обновляет paidAt
   ↓
1. Payment Service уведомляет Delivery Service (через Kafka, topic: paymant.payment-status, статус: completed)
   ↓
2. Delivery Service занимается доставкой товара
```
