Основные компоненты (микросервисы)

Логистическая платформа обычно включает следующие функциональные блоки:

### Архитектура (8 сервисов)

| Сервис                   | База данных        | Задачи                                                        | Взаимодействие                                                                                                         |
| ------------------------ | ------------------ | ------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **API Gateway**          | —                  | Маршрутизация, Keycloak                                       | HTTP: `Gateway` → все сервисы                                                                                          |
| **User Service**         | PostgreSQL + Mongo | Роли (клиент/водитель/админ), профиль (+аватар, документы)    | HTTP: (синхронно): `Getway` → `User`                                                                                   |
| **Transport Service**    | PostgreSQL         | **ТС, водители, статус AVAILABLE/IN_USE**                     | HTTP: `Delivery` → `Transport`;  Kafka: `Transport` → `Reporting`, `Transport` → `Delivery`                            |
| **Delivery Service**     | PostgreSQL +Redis  | **Грузы, доставки, статусы**                                  | HTTP: `Delivery` → `Transport`; Kafka: `Transport` → `Delivery`, `Delivery` → `Reporting`, `Delivery` → `Notification` |
| **Paymant Service**      | PostgreSQL +Redis  | **Расчёт стоимости, оплата, отмена, история платежей**        | HTTP `Delivery`→ `Paymant`; Kafka: `Paymant` → `Reporting`                                                             |
| **Notification Service** |                    | Читает Kafka `delivery.events`, отправляет email/sms/telegram | Kafka (асинхронно)                                                                                                     |
| **Reporting Service**    | PostgreSQL         | Читает/ логирует Kafka -события. Предоставляет отчетности     | Kafka (асинхронно)                                                                                                     |
| **Monitoring Service**   | —                  | Healthchecks (Spring Actuator) + ELK для логов                | HTTP / Actuator                                                                                                        |
