## **Delivery Service**
### Задачи:
- Прием, валидация всей приходящей от заказчика информации;
- Отслеживание статуса заказов;
### Особенности:
- Центральный микросервис системы, управляет почти всеми остальными;
- Высокая нагрузка (собирает почти со всех сервисов);
- Получает уведомления о ЧП с грузом или с ТС, принимает решения о судьбе груза;
- Предоставляет заказчику информацию (счет) за доставку груза на основании финансовой информации с `Paymant Service`;
- Принимает решения о том, когда отправлять груз на основании информации об оплате с `Paymant Service` ;
- Все протоколирует в `Reporting Service`;
### Стек:
- DB: Postgres;
- Kafka;

## **Cargo Service**

### Задачи:
- Вся доступная информация о грузе (хранение информации);
- Управление операциями по приемке и отгрузке;
- Управление складом (WMS): Контроль запасов, размещение товаров;
### Особенности:
- Низкая нагрузка (тут скорее - книжный шкаф);
- Тут информация об упаковке груза, таре, партии, методе складирования...;
- Если происходит ЧП с грузом (порча и т.д...) - то уведомляет сервис доставки;
- Все протоколирует в Reporting Service;
### Стек:
- DB: Postgres;
- Kafka;

## **Delivery Service**. Таблица эндпоинтов *(REST API)*
Все эндпоинты скрыты за Gateway, который проверяет JWT и проксирует запросы.  
Базовый путь: `/api/v1/{service-name}/...`

### **Грузы** (Cargo):

| Метод  | Эндпоинт             | Описание                                     |
| ------ | -------------------- | -------------------------------------------- |
| POST   | `/api/v1/cargo`      | Создать груз                                 |
| GET    | `/api/v1/cargo`      | Список всех грузов (пагинация необязательно) |
| GET    | `/api/v1/cargo/{id}` | Груз по id                                   |
| DELETE | `/api/v1/cargo/{id}` | Удалить груз                                 |

### **Доставки** (Delivery):

| Метод  | Эндпоинт                         | Описание                                | Бизнес-правила                                                        |
| ------ | -------------------------------- | --------------------------------------- | --------------------------------------------------------------------- |
| POST   | `/api/v1/deliveries`             | Создать доставку                        | Проверка веса vs грузоподъёмность; вызов Transport для резервирования |
| GET    | `/api/v1/deliveries`             | Список доставок (фильтр по статусу)     |                                                                       |
| GET    | `/api/v1/deliveries/{id}`        | Детали доставки                         |                                                                       |
| PUT    | `/api/v1/deliveries/{id}/status` | Обновить статус (IN_TRANSIT, DELIVERED) | Запрещено менять CANCELLED                                            |
| DELETE | `/api/v1/deliveries/{id}/cancel` | Отмена доставки                         | Статус → CANCELLED, транспорт → AVAILABLE                             |

### **Платежи** (Paymants):

| Метод | Эндпоинт                     | Описание                                    | Бизнес-правила                                                                                        |
| ----- | ---------------------------- | ------------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| GET   | `/api/v1/paymants/{id}/cost` | Рассчитать стоимость *(просто узнать цену)* | синхронно идет за расчетами в Paymant Service  с ролью SERVICE Базовая_ставка + вес*10 + расстояние*5 |
| POST  | `/api/v1/paymants/bill`      | Сформировать счет на оплату                 | синхронно идет за расчетами в Paymant Service  с ролью SERVICE Базовая_ставка + вес*10 + расстояние*5 |
