# 2. **Сборка**, **артефакты** и **Docker**

---
## 5. Оптимизация Docker-образов

- **Multi-stage Build:** Разделение этапа сборки (JDK + Maven) и этапа запуска (JRE/Distroless). Это уменьшает размер образа в 2–3 раза.
    
- **Базовые образы:** Использование `Distroless` (от Google) или `Eclipse Temurin` на базе Alpine. Distroless безопаснее, так как в нем нет shell и лишних утилит.
    
- **Безопасность:** Инструкция `USER nonroot` обязательна. Приложение не должно работать от root.
    
- **Пример:**
```Dockerfile
FROM eclipse-temurin:21-jdk-alpine AS build
COPY . .
RUN ./mvnw package -DskipTests

FROM gcr.io/distroless/java21-debian12:nonroot
COPY --from=build /target/*.jar app.jar
ENTRYPOINT ["java", "-jar", "/app.jar"]
```

## 6. Ускорение сборки и кэширование

- **Кэш зависимостей:** CI-система (GitLab/GitHub) должна кэшировать директорию `~/.m2` или `~/.gradle` между запусками, чтобы не скачивать интернет каждый раз.
    
- **Docker Layer Caching:** Сначала копируются только файлы сборки (`pom.xml` / `build.gradle`), запускается скачивание зависимостей, и только потом копируется исходный код. Если зависимости не менялись, Docker возьмет их из кэша.
    
- **Слои:** Чем реже меняется информация в слое, тем выше он должен быть в Dockerfile.

## 7. Версионирование и артефакты

- **SNAPSHOT:** Изменяемая версия для разработки. Не должна попадать в Production.
    
- **Release:** Фиксированная версия. Никогда не перезаписывается в репозитории (Nexus/Artifactory).
    
- **Тегирование:** Рекомендуется использовать **Git Commit Hash** для каждого билда и **SemVer** (1.2.3) для релизов. Это обеспечивает 100% прослеживаемость кода.

## 8. Неизменяемость артефактов (Immutability)

- **Суть:** Собрали один раз — деплоим везде. JAR-файл или Docker-образ, протестированный на Staging, должен быть идентичен тому, что пойдет на Production.
    
- **Почему это важно:** Исключаются ошибки из-за разницы версий компилятора, окружения сборки или внезапного обновления библиотек в процессе пересборки. Конфигурация меняется извне, артефакт — никогда.
