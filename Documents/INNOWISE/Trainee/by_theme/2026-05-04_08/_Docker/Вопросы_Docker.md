Для позиции Middle Java Developer на техническом интервью Docker обычно рассматривается не как инструмент администрирования, а как часть процесса доставки кода (CI/CD) и среды запуска (Runtime).

Ниже представлен перечень актуальных вопросов, от самых часто встречающихся (база) до более глубоких и специфичных для Middle-уровня.

---
## 1. Базовый фундамент (Спрашивают почти всегда)

1. **Разница между Docker Image и Docker Container.** Понимание образа как неизменяемого шаблона и контейнера как запущенного процесса.
    
2. **Dockerfile: разбор основных инструкций.** FROM, COPY vs ADD, RUN, CMD vs ENTRYPOINT, EXPOSE, WORKDIR. Почему COPY предпочтительнее ADD?
    
3. **Разница между ENTRYPOINT и CMD.** Как они комбинируются для запуска `java -jar` и возможности переопределения параметров при запуске.
    
4. **Docker Compose.** Запуск связки Spring Boot + БД, описание сервисов, сетей и томов. Разница между `depends_on` и реальной готовностью базы (healthcheck + wait-for-it.sh).
    
5. **Слои (Layers) и кэширование.** Как Docker кэширует слои, почему важен порядок команд в Dockerfile и как не "инвалидировать" кэш раньше времени при сборке Java-приложения.
    
6. **Управление переменными окружения.** Как прокидывать `SPRING_PROFILES_ACTIVE`, конфиги БД или параметры JVM (`-Xmx`, `JAVA_OPTS`) через `-e` в CLI или `environment` в Compose.
    
7. **Проброс портов (Port mapping).** Разница между `EXPOSE` в Dockerfile и публикацией портов `-p` (Publish), а также взаимодействие контейнеров внутри одной сети по именам сервисов.
    

## 2. Специфика Spring Boot и Java (Middle-уровень)

8. **Оптимизация размера образа.** Выбор базового образа (JRE вместо JDK, Alpine, Distroless, Eclipse Temurin). Использование `.dockerignore`.
    
9. **Multi-stage builds.** Сборка проекта (Maven/Gradle) в одном контейнере и копирование только JAR-файла в финальный минимальный образ.
    
10. **Spring Boot Layered JAR.** Использование `-Djarmode=layertools` для разделения зависимостей и кода приложения по разным слоям Docker для ускорения деплоя.
    
11. **Volumes vs Bind Mounts.** Как сохранять логи, файлы или данные БД. Решение проблемы потери данных при перезапуске контейнера.
    
12. **Resource Limits и JVM.** Как ограничения `--memory` и `--cpus` влияют на Java (особенно в старых версиях Java) и что такое OOM Killer.
    
13. **Healthchecks.** Разница между `HEALTHCHECK` в Dockerfile и Spring Boot Actuator. Как оркестратор понимает, что приложение "прогрелось".
    
14. **Логирование.** Работа с `stdout/stderr` вместо записи в файлы внутри контейнера. Команды `docker logs` и их параметры.

## 3. Эксплуатация и Безопасность

15. **Безопасность (Root user).** Почему запуск от root — это риск, и как использовать инструкцию `USER` в Dockerfile для запуска Java-процесса.
    
16. **Проблема PID 1 и Graceful Shutdown.** Почему Java может не ловить сигналы завершения (SIGTERM) и зачем нужны инициализаторы процесса вроде `tini` или флаг `--init`.
    
17. **Docker Network Drivers.** Разница между `bridge`, `host` и `none`. В каких случаях оправдано использование `host`-сети.
    
18. **Диагностика и дебаг.** Использование `docker exec -it` для входа в контейнер, `docker inspect` для просмотра метаданных и удаленная отладка (проброс порта 5005 для JDWP).
    
19. **Очистка ресурсов.** Работа с "осиротевшими" ресурсами (dangling images/volumes) и использование команды `docker system prune`.

## 4. Концептуальные и редкие вопросы

20. **Разница между Docker и VM.** Контейнеризация на уровне ядра ОС против виртуализации железа.
    
21. **CI/CD и реестры образов.** Процесс push/pull в Docker Hub или Nexus, роль Docker в пайплайне.
    
22. **Docker socket mount (DinD).** Проброс `/var/run/docker.sock` внутрь контейнера — когда это нужно (например, для Jenkins агентов) и почему это небезопасно.
    
23. **Масштабирование.** Разница между Docker Compose, Swarm и Kubernetes (на уровне концепций: зачем переходить с Compose на K8s).
    
24. **BuildKit.** Современный движок сборки, его преимущества и параллелизация этапов.
    
25. **Сохранение образов.** Разница между `docker save/load` (экспорт образа) и `docker export/import` (экспорт файловой системы контейнера)..

---

**Совет для Middle:** На интервью по Docker для Java-разработчика чаще всего "копают" в две стороны: **оптимизация сборки** (чтобы CI/CD не длился вечность) и **взаимодействие с инфраструктурой** (сети, тома, лимиты памяти). Уделите этим темам особое внимание.
