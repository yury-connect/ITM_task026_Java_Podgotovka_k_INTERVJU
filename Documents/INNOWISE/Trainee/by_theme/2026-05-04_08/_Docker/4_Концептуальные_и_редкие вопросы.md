# 4. Концептуальные и редкие вопросы

---
## 20. Разница между Docker и VM

- **Виртуальная машина:** Эмулирует «железо», запускает полную гостевую ОС со своим ядром. Высокие накладные расходы на ресурсы.
    
- **Контейнер:** Использует ядро хоста. Изолирует процессы с помощью `namespaces` (что видит процесс) и `cgroups` (сколько ресурсов потребляет). Контейнеры запускаются за секунды и потребляют в разы меньше RAM.

## 21. CI/CD и Реестры (Registry)

Docker-образ — это единица поставки (artifact).

- Пайплайн обычно выглядит так: `Build (Java) -> Docker Build -> Docker Push (Registry) -> Deploy`.
    
- Популярные реестры: **Docker Hub**, **Harbor** (self-hosted), **Amazon ECR**, **Nexus Container Registry**.

## 22. Docker socket mount (DinD)

Монтирование `/var/run/docker.sock` внутрь контейнера позволяет Docker-клиенту внутри контейнера отдавать команды Docker-демону хоста.

- **Зачем:** Чтобы Jenkins в контейнере мог собирать и запускать другие Docker-контейнеры.
    
- **Опасность:** Это "дыра" в безопасности. Контейнер с доступом к сокету может легко захватить контроль над всем сервером.

## 23. Масштабирование: Compose vs Swarm vs Kubernetes

- **Compose:** Инструмент для запуска группы контейнеров на **одной** машине. Нет автохилинга (самолечения) и сложного масштабирования.
    
- **Swarm:** Родной инструмент Docker для кластеров. Проще в настройке, чем K8s, но менее гибкий.
    
- **Kubernetes (K8s):** Стандарт индустрии. Умеет автоматически перезапускать упавшие контейнеры, масштабировать их под нагрузкой и управлять сложными сетевыми маршрутами.

## 24. BuildKit

Это современный движок сборки, который ускоряет `docker build`.

- Поддерживает **параллельную сборку** независимых стадий (stages).
    
- Позволяет использовать `SSH агентов` и `secrets` во время сборки, не оставляя их в слоях образа.
    
- Умнее кэширует промежуточные этапы Java-сборки.

## 25. Сохранение образов: save/load vs export/import

- **save/load:** Работает с **образами**. Сохраняет всю историю слоев. Используется для переноса образа на сервер без интернета.
    
- **export/import:** Работает с **контейнерами** (запущенными процессами). Экспортирует только текущее состояние файловой системы в один слой ("схлопывает" слои). Вся история Dockerfile при этом теряется. Почти не используется в обычной Java-разработке.

---

**Итог:** Для успешного прохождения интервью сфокусируйтесь на **PID 1**, **оптимизации слоев для Java** и **сетевом взаимодействии** контейнеров в Compose.
