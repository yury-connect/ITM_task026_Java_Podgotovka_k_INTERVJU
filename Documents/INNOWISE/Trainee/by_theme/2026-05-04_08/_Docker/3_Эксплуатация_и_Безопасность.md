# 3. Эксплуатация и Безопасность

---
## 15. Безопасность и инструкция USER

Запуск Java-приложения от **root** — это критическая уязвимость. Если злоумышленник взломает приложение, он получит права суперпользователя в контейнере, а в некоторых случаях — и доступ к ресурсам хоста.

- **Решение:** В Dockerfile нужно создать системного пользователя и переключиться на него.
    
- **Пример:**
```Dockerfile
RUN groupadd -r spring && useradd -r -g spring springuser
USER springuser
```

Это ограничивает права процесса Java только необходимым минимумом.

## 16. Проблема PID 1 и Graceful Shutdown

В Linux процесс с **PID 1** (init) обязан обрабатывать сигналы завершения и «усыновлять» осиротевшие процессы. Java-машина по умолчанию не ведет себя как полноценный init-система.

- **Проблема:** При `docker stop` Docker посылает `SIGTERM`. Если Java-процесс имеет PID 1 и не настроен специально, он может игнорировать сигнал, и через 10 секунд Docker убьет его жестко (`SIGKILL`). Spring Boot не успеет закрыть соединения и завершить транзакции.
    
- **Решение:** 1. Использовать флаг `--init` в `docker run`. 2. Использовать легковесный init-процесс, например **tini**, который корректно пробрасывает сигналы в JVM. 3. Использовать `exec` форму `ENTRYPOINT ["java", "..."]`, чтобы Java стала основным процессом.

## 17. Сетевые драйверы: bridge, host, none

- **bridge** (по умолчанию): Создает виртуальный мост. Контейнеры изолированы, но могут общаться друг с другом по IP или именам (в рамках одной сети). Требует проброса портов (`-p`).
    
- **host**: Убирает сетевую изоляцию. Контейнер использует порты хоста напрямую. Это дает максимальную производительность (нет накладных расходов на NAT), но небезопасно и создает конфликты портов.
    
- **none**: Сетевой стек отсутствует. Применяется для максимально изолированных вычислений (например, генерация ключей или экспорт тяжелых архивов).

## 18. Диагностика и дебаг (JDWP)

Для Middle-разработчика важно уметь "заглянуть под капот":

- **Инспекция:** `docker inspect` — показывает IP-адрес, монтирования и переменные окружения.
    
- **Интерактив:** `docker exec -it <id> bash` — позволяет проверить наличие файлов или доступность БД через `ping/telnet`.
    
- **Удаленная отладка:** Чтобы подключиться отладчиком из IntelliJ IDEA к контейнеру, нужно пробросить порт (обычно 5005) и передать JVM флаги JDWP через переменные окружения.

## 19. Очистка ресурсов (Prune)

Docker не удаляет старые данные автоматически. "Осиротевшие" (dangling) образы возникают при пересборке с тем же тегом.

- `docker system prune` — удаляет все остановленные контейнеры, неиспользуемые сети и "висячие" образы без тегов.
    
- В CI/CD важно регулярно чистить место, чтобы диск сервера не переполнился слоями от старых билдов.
