#### Основные (Классика бизнеса)
- **B2B** — Business-to-Business (бизнес для бизнеса).    
  > много отчетов, интеграций (SOAP, XML), безопасность, долгая поддержка, скучный интерфейс
- **B2C** — Business-to-Consumer (бизнес для частного лица).   
  >высокие нагрузки (асинхронность, кэш), красивые интерфейсы, мобилки, стартапы 
- **B2G** — Business-to-Government (бизнес для государства, госзакупки).    
- **B2B2C** — Business-to-Business-to-Consumer (бизнес передает продукт другому бизнесу, чтобы тот продал людям).    
#### Управление и сотрудники
- **B2E** — Business-to-Employee (внутренние порталы, сервисы для сотрудников).    
- **B2J (редко)** — Business-to-Job (рынок труда, HR-технологии — в России так говорят редко, чаще H2M или просто рекрутинг).   
#### IT Архитектура и API (Технические)
- **B2B API** — программный интерфейс для интеграции с системами других компаний.   
- **B2C API** — API, которое используют мобильные приложения клиентов.    
- **B2Bi** — Business-to-Business интеграция (подвид с упором на обмен данными между системами).    
#### Иногда путают, но это про другое:
- **B2D** — Business-to-Developer (продажа инструментов и API разработчикам).    
- **B2R** — Business-to-Reseller (бизнес для продажи через посредников/партнеров).

**JPA (Java Persistence API)** — стандарт для работы с базами данных в Java. Позволяет не писать SQL-запросы вручную, а работать с таблицами как с обычными Java-объектами (автоматически сохраняя их в БД). _Упрощает код, но скрывает детали SQL._

**JMS (Java Message Service)** — стандарт для обмена сообщениями между приложениями (очереди задач/уведомления). Одно приложение кладет сообщение, другое его забирает и обрабатывает. _Нужен для асинхронности и надежности (сообщение не потеряется)._

**JAX-WS** — стандарт для **SOAP** веб-сервисов. Общаются по строгому XML-протоколу (контракт обязателен). _Используется в банках/госструктурах, где важна формальность._

**JAX-RS** — стандарт для **REST** веб-сервисов. Общаются через HTTP (как сайты) в формате JSON/XML. _Самый популярный сейчас для создания API (удобно и легко)._

###  ПРОСТЕЙШЕЕ


**Ansible** — инструмент для автоматической настройки серверов (управление конфигурацией). Позволяет одной командой установить софт на 100 машин или обновить конфиги. _Пульт управления для целой армии серверов._

**Maven** — инструмент для сборки Java-проектов и управления зависимостями (библиотеками). Сам качает нужные библиотеки из интернета и собирает проект в готовый .jar/.war файл. _Клава, которая сама приносит продукты и раскладывает по полкам._

**JUnit** — библиотека для написания и запуска тестов в Java. Проверяет, работает ли код так, как задумано. _Экзаменатор для твоего кода._

**Jenkins** — инструмент для автоматической сборки и тестирования проекта (Continuous Integration). Сам собирает код, прогоняет тесты и выкладывает на сервер после каждого изменения. _Робот-строитель, который работает без выходных._

**Git** — система контроля версий. Хранит историю изменений кода, позволяет нескольким разработчикам работать над проектом одновременно и откатывать ошибки. _«Машина времени» для кода._
