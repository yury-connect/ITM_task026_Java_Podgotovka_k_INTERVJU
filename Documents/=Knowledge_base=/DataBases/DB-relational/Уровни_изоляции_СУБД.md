Уровни изоляции транзакций, СУБД-зависимые дефолты и аномалии"

Согласно стандарту ANSI SQL, существует 4 уровня изоляции, которые борются с тремя ключевыми аномалиями: 
- Dirty Read (чтение незакомиченных данных), 
- Non-Repeatable Read (неповторяемое чтение строки внутри одной транзакции) и 
- Phantom Read (появление новых строк в диапазоне при повторном запросе).

1. Read Uncommitted — допускает все аномалии. 
2. Read Committed — защищает от Dirty Read. Является дефолтным уровнем для PostgreSQL. Каждая инструкция внутри транзакции видит новый снимок данных. 
3. Repeatable Read — защищает от Non-Repeatable Read. Является дефолтным уровнем для MySQL (InnoDB). Важно: Реализация Repeatable Read в PostgreSQL основана на снимочной изоляции (Snapshot Isolation) и под капотом предотвращает также и Phantom Read, что жестче требований стандарта.
4. Serializable — гарантирует полную изоляцию, транзакции эквивалентны последовательному выполнению. 

Архитектурно на уровне Read Committed в Postgres блокировка строк не удерживается при чтении, а в Repeatable Read снимок данных фиксируется строго на момент выполнения первой команды транзакции.
