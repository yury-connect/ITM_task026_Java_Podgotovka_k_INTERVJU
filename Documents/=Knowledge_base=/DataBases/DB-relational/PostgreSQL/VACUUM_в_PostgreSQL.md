## 🧹 Виды **VACUUM** в **PostgreSQL** <br>— краткий ответ для собеседования

## 🎯 **Кратко** (*для собеседования*)

#### В PostgreSQL есть **3 вида** VACUUM:
	
1. **Standard VACUUM** — обычная очистка (*не блокирует чтение/запись*)
    
2. **VACUUM FULL** — полная очистка (*блокирует таблицу, возвращает место ОС*)
    
3. **VACUUM FREEZE** — "*заморозка*" старых транзакций (*защита от `wraparound`*)

#### **Главное отличие:**
	
- **Standard** — чистит «мусор» (*dead tuples*) и освобождает место внутри таблицы для переиспользования.
    
- **FULL** — делает то же самое, но **сжимает** таблицу и **возвращает** место операционной системе (но полностью **блокирует** таблицу!).
    
- **FREEZE** — обновляет возраст транзакций, принудительно помечая строки как «замороженные» (не зависит от `vacuum_freeze_min_age`).

---
## 🔬 Чуть **подробнее**

#### 1. **Standard VACUUM** (*Обычная очистка*)
```sql
VACUUM;
VACUUM (VERBOSE, ANALYZE) table_name;
```

|Характеристика|Описание|
|---|---|
|**Что делает**|Помечает dead tuples (мёртвые строки) как свободное место|
|**Блокировка**|❌ Не блокирует (AccessShareLock) — можно читать и писать|
|**Возврат места ОС**|❌ Нет — место остаётся внутри файла таблицы (для переиспользования)|
|**Когда использовать**|Регулярно, в фоне (автовакуум)|
|**Плюсы**|Не мешает работе приложения|
|**Минусы**|Не уменьшает размер файла на диске|

---
#### 2. **VACUUM FULL** (*Полная очистка*)
```sql
VACUUM FULL table_name;
```

| Характеристика         | Описание                                                         |
| ---------------------- | ---------------------------------------------------------------- |
| **Что делает**         | Создаёт новую копию таблицы, <br>переписывая только живые строки |
| **Блокировка**         | ✅ **AccessExclusiveLock** — таблица полностью заблокирована!     |
| **Возврат места ОС**   | ✅ Да — файл таблицы уменьшается                                  |
| **Когда использовать** | Когда нужно вернуть место на диске                               |
| **Плюсы**              | Уменьшает размер таблицы на диске                                |
| **Минусы**             | ❌ Останавливает работу с таблицей на время выполнения            |

**⚠️ Внимание:** Требует места на диске (*создаёт копию таблицы*).

---
#### 3. **VACUUM FREEZE** (*Заморозка*)
```sql
VACUUM FREEZE table_name;
```

| Характеристика         | Описание                                                                                     |
| ---------------------- | -------------------------------------------------------------------------------------------- |
| **Что делает**         | Принудительно «*замораживает*» строки, <br>обновляя `xmin` до значения `FrozenTransactionId` |
| **Блокировка**         | ❌ Как у обычного VACUUM (*не блокирует*)                                                     |
| **Возврат места ОС**   | ❌ Нет                                                                                        |
| **Когда использовать** | При угрозе **transaction wraparound** <br>(*превышение `age` > 2 млрд*)                      |
| **Плюсы**              | Защищает от катастрофического отказа БД                                                      |
| **Минусы**             | Может нагружать I/O                                                                          |

---
## 📊 Сравнительная таблица

|Параметр|VACUUM|VACUUM FULL|VACUUM FREEZE|
|---|---|---|---|
|**Возвращает место ОС**|❌|✅|❌|
|**Блокирует таблицу**|❌|✅ (AccessExclusiveLock)|❌|
|**Можно читать/писать**|✅|❌|✅|
|**Сбрасывает статистику**|❌|❌|❌|
|**Защита от wraparound**|✅ (с `FREEZE`)|❌|✅|
|**Требует доп. места**|❌|✅|❌|
|**Использует автовакуум**|✅|❌|✅ (если настроен)|

---
## ⚙️ `VACUUM FULL` vs `VACUUM` — простая аналогия

| Действие      | Обычный VACUUM                                                                | VACUUM FULL                                                                |
| ------------- | ----------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| **Аналогия**  | Уборка в комнате: <br>складываешь вещи на места, <br>освобождаешь поверхность | Капитальный ремонт: <br>выносишь всё, делаешь ремонт, <br>заносишь обратно |
| **Результат** | Место появилось, но размер комнаты не изменился                               | Комната стала меньше, но на время ремонта никто не может войти             |

## 🔧 **Автовакуум** (*Autovacuum*)

PostgreSQL автоматически запускает **Standard VACUUM** <br>и **ANALYZE** в фоне через параметры:
```sql
-- Настройки автовакуума
autovacuum = on
autovacuum_vacuum_threshold = 50
autovacuum_vacuum_scale_factor = 0.2
autovacuum_analyze_threshold = 50
autovacuum_analyze_scale_factor = 0.1
```

**VACUUM FULL** и **VACUUM FREEZE** не запускаются автоматически, <br>их нужно вызывать вручную!

---
## 🚨 Когда делать VACUUM FULL?

**Только когда:**
	
- Таблица **очень сильно** раздулась (в 2+ раза больше, чем нужно)
    
- Есть **свободное место на диске** (минимум размер таблицы × 2)
    
- Можно **остановить работу** с таблицей (или делать ночью)

```sql
-- Проверить размер таблицы
SELECT pg_size_pretty(pg_total_relation_size('table_name'));
-- Проверить количество dead tuples
SELECT schemaname, tablename, n_dead_tup, n_live_tup
FROM pg_stat_user_tables
WHERE tablename = 'table_name';
```

---
## 💡 Комбинированный подход (рекомендация)
```sql
-- Обычная очистка + обновление статистики (регулярно)
VACUUM (ANALYZE) table_name;
-- Если таблица сильно раздулась → FULL (ночью в downtime)
VACUUM FULL table_name;
-- Если возраст старых транзакций приближается к лимиту → FREEZE
VACUUM FREEZE table_name;
```

---
## 🧠 Итог (для запоминания)

|Что нужно|Команда|
|---|---|
|**Регулярная уборка** (без блокировки)|`VACUUM;`|
|**Вернуть место на диске** (с блокировкой)|`VACUUM FULL;`|
|**Защита от wraparound**|`VACUUM FREEZE;`|

> **Самое важное:** `VACUUM` не возвращает место ОС, `VACUUM FULL` — возвращает, но блокирует таблицу!
