`@Transactional` — это ваш главный инструмент для управления транзакциями в Spring. Он определяет, как и когда должны открываться, коммититься или откатываться транзакции базы данных[](https://www.baeldung.com/spring-transactional-propagation-isolation).

### 🔄 Все режимы Propagation (Поведение транзакций)

Этот параметр определяет, как метод должен реагировать на существующую транзакцию[](https://www.baeldung.com/spring-transactional-propagation-isolation). `REQUIRED` используется по умолчанию[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/2.0.6/javadoc-api/org/springframework/transaction/annotation/Transactional.html)[](https://docs.spring.io/spring-framework/docs/5.3.0/javadoc-api/org/springframework/transaction/annotation/Transactional.html).

- **`REQUIRED`**: Поддерживает текущую транзакцию. Если её нет, создает новую[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/7.0.x/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://www.baeldung.com/spring-transactional-propagation-isolation).
    
- **`SUPPORTS`**: Поддерживает текущую транзакцию. Если её нет, выполняется без транзакции[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/7.0.x/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://www.baeldung.com/spring-transactional-propagation-isolation).
    
- **`MANDATORY`**: Обязательно должен быть вызван в рамках существующей транзакции. Если её нет, бросает исключение[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/7.0.x/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://www.baeldung.com/spring-transactional-propagation-isolation).
    
- **`REQUIRES_NEW`**: Всегда создает **новую** транзакцию. Если есть текущая, она приостанавливается (suspend)[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/7.0.x/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://www.baeldung.com/spring-transactional-propagation-isolation).
    
- **`NOT_SUPPORTED`**: Выполняется **без** транзакции. Если есть текущая, она приостанавливается[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/7.0.x/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://www.baeldung.com/spring-transactional-propagation-isolation).
    
- **`NEVER`**: Выполняется **без** транзакции. Если есть текущая, бросает исключение[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/7.0.x/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://www.baeldung.com/spring-transactional-propagation-isolation).
    
- **`NESTED`**: Выполняется во **вложенной** транзакции, если есть текущая (использует savepoint). Если нет — ведет себя как `REQUIRED`[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/7.0.x/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://www.baeldung.com/spring-transactional-propagation-isolation).
    

> **Важные нюансы**:
> 
> 1. **Приостановка (`suspend`)**: Режимы `REQUIRES_NEW` и `NOT_SUPPORTED` приостанавливают текущую транзакцию. Эта функция зависит от конкретного менеджера транзакций и может работать не везде (например, требует настройки `JtaTransactionManager`)[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/7.0.x/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/2.0.7/javadoc-api/org/springframework/transaction/annotation/Propagation.html#REQUIRED).
>     
> 2. **Вложенность (`NESTED`)**: Реально работает только с `DataSourceTransactionManager` (JDBC). Для JTA требуется поддержка со стороны провайдера[](https://docs.spring.io/spring-framework/docs/5.3.10/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://docs.spring.io/spring-framework/docs/7.0.x/javadoc-api/org/springframework/transaction/annotation/Propagation.html)[](https://www.baeldung.com/spring-transactional-propagation-isolation).
>     

### 💡 Коротко о главном

- **Настройки `isolation`, `timeout` работают только для новых транзакций**: Если вы выставите высокий уровень изоляции на методе с `propagation = Propagation.REQUIRED`, но он будет вызван из другого метода, уже открывшего транзакцию, ваша настройка будет проигнорирована[](https://docs.spring.io/spring-framework/docs/5.3.0/javadoc-api/org/springframework/transaction/annotation/Transactional.html).
    
- **`readOnly` - это подсказка**: Это не жесткая гарантия. База данных и Hibernate могут использовать этот флаг для оптимизации, но не всегда блокируют запись[](https://docs.spring.io/spring-framework/docs/5.3.0/javadoc-api/org/springframework/transaction/annotation/Transactional.html)[](https://www.baeldung.com/spring-vs-jta-transaction
