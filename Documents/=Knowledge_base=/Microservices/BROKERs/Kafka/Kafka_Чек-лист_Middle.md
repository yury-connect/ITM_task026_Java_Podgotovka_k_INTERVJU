## 📚 Kafka: Чек-лист *Middle*-разработчика

---
### 🔴 **Must-have** (знать наизусть, отвечать без подготовки)

| №   | Понятие                                         | Суть                                          |
| --- | ----------------------------------------------- | --------------------------------------------- |
| 1   | **Topic**                                       | Канал/очередь сообщений                       |
| 2   | **Partition**                                   | Часть топика, единица параллелизма            |
| 3   | **Offset**                                      | Номер сообщения в партиции                    |
| 4   | **Consumer Group**                              | Группа консьюмеров, делящих партиции          |
| 5   | **Producer**                                    | Отправляет сообщения                          |
| 6   | **Consumer**                                    | Читает сообщения                              |
| 7   | **Broker**                                      | Сервер Kafka                                  |
| 8   | **Leader / Follower / ISR**                     | Механизм репликации                           |
| 9   | **Acks**                                        | `0`, `1`, `all` — гарантия доставки           |
| 10  | **Rebalancing**                                 | Перераспределение партиций между консьюмерами |
| 11  | **Offset commit**                               | Сохранение позиции чтения                     |
| 12  | **At-least-once / At-most-once / Exactly-once** | Семантики доставки                            |

### 🟠 **Важно* (часто спрашивают, нужно уверенно объяснять)*

| №   | Понятие                                             | Суть                                                |
| --- | --------------------------------------------------- | --------------------------------------------------- |
| 13  | **Key**                                             | Определяет партицию (`hash(key) % partitions`)      |
| 14  | **RoundRobin / Range / Sticky / CooperativeSticky** | Стратегии назначения партиций                       |
| 15  | **Consumer Group Coordinator**                      | Брокер, управляющий группой                         |
| 16  | **__consumer_offsets**                              | Внутренний топик с offset-ами групп                 |
| 17  | **Heartbeat / Session timeout**                     | Механизм обнаружения "мёртвых" консьюмеров          |
| 18  | **Idempotent producer**                             | `enable.idempotence=true` — защита от дублей        |
| 19  | **Transaction**                                     | Exactly-once для producer-consumer связки           |
| 20  | **Replication factor**                              | Количество копий партиции                           |
| 21  | **min.insync.replicas**                             | Минимум реплик для успешной записи (при `acks=all`) |

## 🎯 Приоритетная карта обучения

```text
Блок 1 (20% времени) → ТЕОРИЯ
├── Topic / Partition / Offset
├── Producer / Consumer / Consumer Group
├── Broker / Leader / ISR
└── Acks / Replication

Блок 2 (50% времени) → ПРАКТИКА
├── Написать producer + consumer
├── Поработать с consumer group (несколько консьюмеров)
├── Потрогать rebalancing (добавить/удалить консьюмера)
├── Поиграть с acks (потеря сообщений)
└── Коммит offset (авто/ручной)

Блок 3 (20% времени) → УГЛУБЛЕНИЕ
├── Idempotent producer / Transactions
├── Стратегии назначения партиций (Range, RoundRobin, Sticky)
├── Cooperative vs Eager ребалансировка
└── Координатор группы + __consumer_offsets

Блок 4 (10% времени) → АРХИТЕКТУРА
├── Kafka Streams (если нужно)
├── Kafka Connect (если нужно)
└── KRaft (новое поколение без ZooKeeper)
```

## ✅ Проверка себя

> **Middle уверенно отвечает на вопросы из красного и оранжевого блоков.**
> 
> **Знает, как настроить producer/consumer под разные семантики доставки.**
> 
> **Понимает, что происходит при ребалансировке и как её оптимизировать.**
