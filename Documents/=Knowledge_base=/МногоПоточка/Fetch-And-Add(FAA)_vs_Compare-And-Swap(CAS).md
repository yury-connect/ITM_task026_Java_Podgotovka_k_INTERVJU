# Fetch-And-Add (*FAA*)

---
> [!IMPORTANT]
> КРАТКО
### СУТЬ (*для конспекта*)

**Fetch-And-Add (FAA)** — атомарная операция, которая:
	
1. **Читает** текущее значение переменной
    
2. **Прибавляет** к нему заданное число
    
3. **Записывает** новое значение
    
4. Возвращает **старое** значение

**Ключевое:** всё это происходит **одной неделимой** инструкцией на уровне процессора.

---
> [!TIP]
> ПОДРОБНЕЕ
### ДЕТАЛИ

**Где используется:**
- Счетчики (`AtomicLong.incrementAndGet()` внутри исп-т FAA на нек-х платформах)
- Очереди (генерация индексов)
- Распределенные системы (генерация ID)

**В Java:**
```java
// Аналог FAA через AtomicLong
AtomicLong counter = new AtomicLong(0);
long oldValue = counter.getAndIncrement(); // возвращает 0, становится 1
// Это и есть Fetch-And-Add
```

**Реализация в процессорах:**
- x86: инструкция `XADD`
- Реализуется без блокировок (lock-free)

**Проблемы:**
- Только для **сложения/вычитания** (не для произвольных операций)
- При переполнении — поведение зависит от архитектуры

---
### СРАВНЕНИЕ: **FAA** vs **CAS**

| Характеристика         | **Fetch-And-Add**                       | **Compare-And-Swap**                                                  |
| ---------------------- | --------------------------------------- | --------------------------------------------------------------------- |
| **Суть**               | Просто <br>прибавить число              | Изменить, только если <br>значение НЕ изменилось <br>с момента чтения |
| **Возвращает**         | Старое значение                         | Булевый флаг (*успех/неуспех*) <br>или старое значение                |
| **Где применяется**    | Счетчики, <br>генераторы номеров        | Сложные структуры данных <br>(стэки, очереди, map'ы)                  |
| **Главный минус**      | Только арифметика                       | Нужен цикл при неудаче <br>(retry-loop)                               |
| **В Java**             | `AtomicLong`<br>   `.getAndIncrement()` | `AtomicReference`<br>   `.compareAndSet()`                            |
| **Производительность** | Быстрее <br>(*одна инструкция*)         | Медленнее <br>(*цикл + сравнение*)                                    |

| FAA                                             | CAS                                          |
| ----------------------------------------------- | -------------------------------------------- |
| Всегда изменяет значение                        | Меняет, только если текущее равно ожидаемому |
| Не может провалиться <br>(*кроме переполнения*) | Может провалиться → нужен retry-цикл         |
| Используется для счётчиков                      | Используется для сложных обновлений          |

---
**Пример для наглядности:**
```java
// FAA (просто +1)
atomicLong.getAndIncrement(); // всегда увеличит

// CAS (с проверкой)
atomicInteger.compareAndSet(expected, newValue); // увеличит, только если никто не изменил
```

```java
AtomicInteger counter = new AtomicInteger(0);
// Аналог FAA:
int old = counter.getAndIncrement(); // old = 0, теперь counter = 1
int current = counter.incrementAndGet(); // теперь counter = 2, current = 2
```

---
**Итог (*запомнить*):**

> **FAA** — для простых счетчиков (*быстро, без циклов*).  
> **CAS** — для сложных операций (*гибко, но с retry*).  
> В современных JVM `AtomicLong.incrementAndGet()` — это CAS в цикле, но на некоторых архитектурах JIT-компилятор заменяет это на аппаратный FAA.

---
**Для конспекта (*одной строкой*):**

> **FAA** — аппаратная атомарная операция "*`прочитал`-`прибавил`-`записал`-`вернул`*". 
> В Java — это `AtomicInteger.getAndIncrement()`. Работает быстрее *локов*, не вызывает *deadlock*. Относится к многопоточности и *lock-free* алгоритмам.

---
**Если спросят на собесе:**  
Скажите, что это примитив для атомарных счётчиков, реализованный через процессорные инструкции, и в *Java* его обёртка — `AtomicInteger`. Главное — не путать с CAS (*это другая операция*).
