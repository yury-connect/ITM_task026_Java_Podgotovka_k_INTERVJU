Когда тебя спросят: _«Что такое процесс и поток? Чем они отличаются?»_, ты можешь выдать такую структуру:

> **Ответ:**
> 
> _«**Процесс** — это экземпляр программы, который выполняется в изолированном адресном пространстве. У него есть своя память, файловые дескрипторы, и он не может напрямую трогать память другого процесса — для общения нужно использовать IPC (сокеты, очереди, файлы)._
> 
> _**Поток** — это легковесный исполнитель внутри процесса. Все потоки одного процесса разделяют общую память (Heap) и метаданные классов, но у каждого есть свой стек (Stack) для локальных переменных и цепочки вызовов._
> 
> _В Java используется нативная потоковая модель — каждый Java-поток маппится на поток ОС. Но с Java 21 появились виртуальные потоки, которые управляются JVM и не привязаны 1:1 к ОС._
> 
> _Из-за того, что потоки разделяют память, возникают классические проблемы многопоточки: гонки данных, дедлоки и проблемы видимости. Их решают через синхронизацию.»_

| Характеристика            | Процесс                                     | Поток                                                     |
| ------------------------- | ------------------------------------------- | --------------------------------------------------------- |
| **Адресное пространство** | ✅ Изолированное                             | ❌ Общее (разделяют Heap)                                  |
| **Свой стек**             | ✅ Есть                                      | ✅ Есть                                                    |
| **Разделяют Heap**        | ❌ Нет                                       | ✅ Да                                                      |
| **Создание**              | Дорогое (копирование ресурсов)              | Дешёвое (только стек + регистры)                          |
| **IPC / синхронизация**   | Сложная (сокеты, файлы, очереди)            | Простая (через общие объекты, `synchronized`)             |
| **Сбой**                  | Падение процесса не убивает другие процессы | Падение потока может убить весь процесс (если не поймать) |
