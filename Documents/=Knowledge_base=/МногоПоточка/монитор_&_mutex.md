## 📌 Что такое монитор? (определение)

**Монитор** — это **механизм синхронизации**, который гарантирует, что **в критическую секцию кода** в один момент времени может войти только **один поток**.

Он обеспечивает:
1. **Взаимное исключение (*mutual exclusion*)** — только один поток выполняет защищенный код.
    
2. **Координацию потоков** — возможность потоков ждать определенных условий (`wait/notify`).

В классической теории монитор содержит:
	
- **Mutex (*блокировку*)** — для эксклюзивного доступа.
    
- **Очередь ожидания (*entry set*)** — потоки, которые хотят захватить монитор.
    
- **Очередь условий (*wait set*)** — потоки, которые вызвали `wait()` и ждут уведомления.

---
## 🔐 Как монитор реализован в Java?

> **В Java каждый объект является монитором.** Да-да, абсолютно любой объект в Java (включая массивы) имеет встроенный монитор.

Это стало возможным благодаря тому, что у каждого объекта в заголовке (объектном заголовке) есть специальное поле под монитор.

### Ключевые элементы:

|Элемент|Описание|
|---|---|
|**`synchronized`**|Ключевое слово, которое позволяет захватить монитор объекта.|
|**Mutex (блокировка)**|Встроен в `Object`. Может быть "свободен" или "захвачен" конкретным потоком.|
|**Entry Set (очередь входа)**|Потоки, которые пытаются войти в `synchronized`-блок, но не могут, потому что монитор занят.|
|**Wait Set (очередь ожидания)**|Потоки, которые вызвали `wait()` на этом объекте и ждут `notify()`/`notifyAll()`.|

## ⚙️ Как это работает (внутренности)

Твой пример с `id` — очень хорошая аналогия. Я ее немного уточню:

> Можно представить монитор как **комнату с замком**:
> 
> - **Замок свободен (`id = 0`)** — любой поток может войти и повесить свой `id` на замок.
>     
> - **Замок занят (`id = 123`)** — другие потоки становятся в **очередь у двери (Entry Set)** и ждут, пока поток 123 выйдет.
>     
> - Если поток внутри комнаты вызвал **`wait()`** — он **отпускает замок** и переходит в другую очередь (**Wait Set**) до тех пор, пока кто-то не вызовет `notify()`/`notifyAll()`.
>

## 🧠 Важные детали (для собеседования)

|Деталь|Объяснение|
|---|---|
|**Монитор есть у каждого объекта**|В Java **нельзя** создать объект без монитора. Это вшито в архитектуру JVM.|
|**`synchronized` бывает на методах и блоках**|На методе — захватывает `this` (или `класс.class` для статического). На блоке — захватывает любой указанный объект.|
|**Монитор реентерабельный**|Если поток уже держит монитор, он может войти в другой `synchronized`-блок на том же объекте (счетчик захвата увеличивается).|
|**`wait()` отпускает монитор**|В отличие от `sleep()` или `yield()`, которые не отпускают блокировки.|
|**`notify()` vs `notifyAll()`**|`notify()` будит один случайный поток из Wait Set. `notifyAll()` — все потоки. На практике `notifyAll()` безопаснее, чтобы избежать "потерянных пробуждений".|

## 🎤 Скрипт для собеседования (короткая версия)

> _«Монитор — это механизм синхронизации, который обеспечивает взаимное исключение и координацию потоков. В Java каждый объект является монитором благодаря встроенному mutex'у в заголовке объекта. Работа с монитором осуществляется через ключевое слово `synchronized`, которое захватывает монитор объекта и не дает другим потокам войти в критическую секцию. Если монитор занят, поток встает в очередь Entry Set. Внутри `synchronized`-блока можно вызвать `wait()` — это отпустит монитор и переведет поток в Wait Set до прихода `notify()` или `notifyAll()`. Важно помнить, что монитор в Java реентерабельный — если поток уже держит монитор, он может войти в другой `synchronized`-блок на том же объекте.»_

## 💡 Бонус (для глубины)

Если интервьюер спросит: _«А где именно в объекте хранится монитор?»_, можно ответить:

> _«В объектном заголовке (object header) есть специальное поле — **mark word**. В нем хранится информация о хэш-коде, GC-статусе и, самое главное, ссылка на монитор, если объект используется как блокировка. JVM использует оптимизации: если монитор не используется, он не выделяется (это называется **biased locking** и **thin locking**). При конкуренции монитор "раздувается" (inflate) до полноценного тяжелого монитора (heavyweight).»_
