**`CountDownLatch`** и **`CyclicBarrier`** - это **синхронизаторы потоков** из пакета `java.util.concurrent`. Они нужны, чтобы **координировать работу неск-х потоков**.

> Это **инструменты для общения потоков между собой**, чтобы они:
> - ждали друг друга
> - сигнализировали о завершении работы
> - синхронизировались в определённых точках

---
## **CountDownLatch** — "защёлка с обратным отсчётом"
Представьте **дверь, кот-я откр-ся только после того, как кто-то нажал кнопку N раз**.
- Создаётся с числом (*например, 3*)
- Потоки вызывают `countDown()` — это как "нажать кнопку" (счётчик уменьшается)
- Когда счётчик доходит до 0 — **дверь открывается** и все ждущие потоки продолжают работу
**Зачем:** Один поток ждёт, пока несколько других потоков завершат свои задачи.

## **CyclicBarrier** — "круговая ограда" / "барьер"
**Точка сбора**, где все участники должны собраться, прежде чем идти дальше.
- Создаётся с числом участников (например, 3)    
- Каждый поток подходит к барьеру и вызывает `await()` — **встаёт в очередь**
- Когда все 3 потока собрались — **барьер ломается** и все одновременно идут дальше
**Зачем:** Несколько потоков должны достичь одной точки, а потом вместе продолжить.

---
# Кратко **о различиях**

### **CountDownLatch** (*Одноразовый*)

**Суть:** Один поток ждёт, пока **N других потоков** выполнят свою работу.

**Как работает:**
- Создаётся счётчик: `new CountDownLatch(3)`
- Каждый поток вызывает `countDown()` (*уменьшает счётчик*)
- Ждущий поток вызывает `await()` — блокируется, пока счётчик не станет 0

**Особенности:**
- ❌ Нельзя переиспользовать (*счётчик не сбрасывается*)
- ✅ Потоки могут выполнять разные задачи
- ✅ Ждать могут несколько потоков одновременно
```java
CountDownLatch latch = new CountDownLatch(3);
	// В 3-х потоках:
latch.countDown();  // выполнили свою часть
	// В главном потоке:
latch.await();  // ждём, пока все 3 не завершат
```

### **CyclicBarrier** (*Многоразовый*)

**Суть:** **N потоков** встречаются в одной точке и ждут ДРУГ ДРУГА.

**Как работает:**
- Создаётся барьер: `new CyclicBarrier(3)`
- Каждый поток вызывает `await()` — блокируется
- Когда все 3 потока вызвали `await()`, барьер "ломается" и **все продолжают работу одновременно**

**Особенности:**
- ✅ Можно переиспользовать (сбрасывается автоматически)    
- ✅ Есть `Runnable`-действие, которое выполняется после сбора всех потоков
- ❌ Все потоки должны выполнять одну и ту же стадию
```java
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("Все собрались, начинаем!");
});
	// В каждом из 3-х потоков:
barrier.await();  // ждём остальных
	// после того как все 3 собрались — продолжаем
```

## Главное отличие

|Характеристика|CountDownLatch|CyclicBarrier|
|---|---|---|
|**Кто кого ждёт**|Один поток ждёт многих|Многие ждут друг друга|
|**Переиспользование**|❌ Нет|✅ Да (reset)|
|**Счётчик**|Уменьшается|Сбрасывается|
|**Действие после сбора**|❌|✅ `Runnable`|
