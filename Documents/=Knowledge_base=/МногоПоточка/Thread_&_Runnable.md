### 1. В чем разница (база)
	
- **`Thread`** — это **класс**, представляющий собой **объект-надстройку над нативным потоком ОС**. Он содержит в себе методы для управления потоком: `start()`, `sleep()`, `interrupt()`, `join()`, `isAlive()` и т.д.
    
- **`Runnable`** — это **функциональный интерфейс** (с единственным методом `run()`), который описывает **абстракцию задачи (work unit)**, которую нужно выполнить в потоке. Он ничего не знает про управление потоком.

---
### 2. Ключевое различие *(самое важное!)*

> **`Thread` — это "кто запускает". `Runnable` — это "что запускают".**

| Аспект                | `Thread`                                                    | `Runnable`                                    |
| --------------------- | ----------------------------------------------------------- | --------------------------------------------- |
| **Суть**              | Сущность потока (исполнитель)                               | Контракт на выполнение кода (задача)          |
| **Тип**               | Класс                                                       | Функциональный интерфейс                      |
| **Методы**            | Много: `start()`, `sleep()`, <br>`join()`, `interrupt()`... | Один: `run()`                                 |
| **Ресурсы**           | Связан с нативным <br>потоком ОС                            | Это просто обычный объект в куче              |
| **Наследование**      | Мешает <br>(занимает extends)                               | Не мешает (можно extends другой класс)        |
| **Переиспользование** | Сложно (каждый раз новый объект)                            | Легко (одну задачу можно отдать разным пулам) |

---
### 3. Когда использовать `Thread`, а когда `Runnable`?

| Вариант                                                                     | Выбор                             | Почему                                                                                                                                                                                                                                                |
| --------------------------------------------------------------------------- | --------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Создание кастомного <br>потока с <br>расширенной логикой <br>управления** | ✅ `extends Thread`                | Если тебе нужно переопределить не только `run()`, но и другие методы (например, `interrupt()` или логику инициализации). Но это **редкий кейс**.                                                                                                      |
| **Обычная задача, которую нужно выполнить в фоне**                          | ✅ `implements Runnable`           | **Вариант по умолчанию.** Потому что:<br>1. Не тратится единственное место под наследование.<br>2. Логика задачи отделена от логики управления потоком.<br>3. Одну и ту же `Runnable` можно передать в несколько потоков или пул (`ExecutorService`). |
| **Работа с `ExecutorService` <br>(пул потоков)**                            | ✅ `Runnable` <br>(или `Callable`) | Пул принимает именно задачи (`Runnable`/`Callable`), а не потоки. Ты не можешь передать `Thread` в `execute()`.                                                                                                                                       |
| **Код, который должен вернуть результат**                                   | ❌ `Runnable` <br>не подходит      | Используй **`Callable<V>`** (возвращает результат и может бросить checked exception).                                                                                                                                                                 |

---
### 4. Самый частый **баг** новичков <br>(*и что обязательно сказать на собеседовании*)

> **⚠️ ВАЖНО:** У `Thread` есть два метода:
> 	
> - `run()` — содержит код, который должен выполниться в потоке.
>     
> - `start()` — запускает **новый нативный поток ОС**, который вызывает `run()`.

#### **Запомни раз и навсегда:**

> **Если вызвать `thread.run()` — код вып-ся в ТЕКУЩЕМ потоке (*без созд. нового*).**  
> **Если вызвать `thread.start()` — создастся НОВЫЙ поток и в нем выполнится `run()`**

Этот нюанс — **любимый вопрос** на собеседованиях! Ты должен его знать и уметь объяснить.

---
### 5. Примеры кода (для иллюстрации)

#### ✅ Правильный подход (Runnable + ExecutorService)
```java
Runnable task = () -> System.out.println("Hello from " + Thread.currentThread().getName());

// Можно запустить вручную
new Thread(task).start();

// А можно в пуле (гибкость!)
ExecutorService pool = Executors.newFixedThreadPool(10);
pool.execute(task);
```

#### ❌ Новичковый баг
```java
Thread t = new Thread(() -> System.out.println("Hello"));
t.run();  // ❌ Будет выполнен в main, а не в новом потоке!
```

### 6. Что еще можно добавить (для уровня Strong Middle)

Если хочешь **добить** вопрос и показать глубину, скажи:

> _«Современная Java предпочитает использовать `Runnable` и `Callable` вместе с `ExecutorService`, а не создавать `Thread` вручную. Это отделяет логику задачи от инфраструктуры потоков, упрощает управление ресурсами (пулы ограничивают число потоков) и облегчает тестирование. Кроме того, с приходом **Virtual Threads** (Java 21) использование `Thread` напрямую почти полностью уходит в прошлое — вместо `new Thread().start()` мы пишем `Thread.startVirtualThread()` или используем `Executors.newVirtualThreadPerTaskExecutor()`, но задача все равно описывается через `Runnable` или `Callable`.»_

## 🎤 Скрипт для собеседования (короткая версия)

> _«Thread — это класс, надстройка над нативным потоком ОС. Runnable — это функциональный интерфейс, представляющий задачу. Главное отличие: Thread умеет управлять потоком (start, sleep, join), а Runnable — только содержит код для выполнения. На практике я всегда использую Runnable, потому что: а) не занимаю наследование, б) одну задачу можно передать в разные потоки или пулы, в) это чище архитектурно. Thread я использую только если нужно переопределить поведение самого потока — но такие кейсы крайне редки. И главное: никогда не путать run() и start() — run() выполняет код в текущем потоке, start() создает новый поток ОС.»_
