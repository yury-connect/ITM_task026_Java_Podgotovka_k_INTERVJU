**Семафор (Semaphore)** — это механизм синхронизации, который ограничивает **количество потоков**, одновременно имеющих доступ к ресурсу, с помощью счетчика разрешений (permits).

---
### 🔧 Как реализован в Java:
	
- Класс **`java.util.concurrent.Semaphore`**.
    
- В конструкторе задается количество разрешений (например, `new Semaphore(5)`).
    
- **`acquire()`** — запросить разрешение (если счетчик > 0, забираем; если 0 — блокируемся).
    
- **`release()`** — вернуть разрешение (счетчик увеличивается).

---
### 📊 Пример (пул из 3 подключений к БД):
```java
Semaphore pool = new Semaphore(3); // только 3 потока могут работать
void useConnection() {
    pool.acquire();  // запросил разрешение (счетчик -1)
    try {
        // работа с БД
    } finally {
        pool.release(); // вернул разрешение (счетчик +1)
    }
}
```

---
### 🆚 Чем отличается от `synchronized`?

|                      | `synchronized`            | `Semaphore`                             |
| -------------------- | ------------------------- | --------------------------------------- |
| **Доступ**           | Только 1 поток            | **N потоков** (задаётся числом)         |
| **Где используется** | Защита критической секции | Ограничение ресурсов <br>(пулы, лимиты) |
| **"Честность"**      | Нет                       | Можно включить (fair)                   |

### 🎤 Для собеседования (одной фразой):

> _«Семафор в Java — это синхронизатор с счетчиком разрешений. Он позволяет ограничить число потоков, работающих с ресурсом одновременно (например, 10 подключений к БД). Поток вызывает `acquire()`, чтобы получить разрешение, и `release()`, чтобы освободить его. Если разрешения закончились, поток блокируется.»_
