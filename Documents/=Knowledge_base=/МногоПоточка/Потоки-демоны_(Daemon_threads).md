**Потоки-демоны (Daemon threads)** — это фоновые потоки, которые **не мешают JVM завершиться**. Если в приложении остались только потоки-демоны, JVM **завершается**, не дожидаясь их окончания.

---
## 🎯 Для чего они нужны?

- Для выполнения **вспомогательных/сервисных задач**, которые должны работать в фоне, но не являются критичными для работы приложения.
    
- Классические примеры:
    
    - **Сборщик мусора (GC)**.
        
    - **Finalizer**.
        
    - Фоновый мониторинг, логирование, очистка кэша.
        

---
## ⚙️ Как создать поток-демон?
```java
Thread daemonThread = new Thread(() -> {
    while (true) {
        // бесконечная фоновая работа
    }
});

daemonThread.setDaemon(true); // сделать демоном ДО старта!
daemonThread.start();
```

**Важно:** `setDaemon(true)` нужно вызывать **до** `start()`, иначе бросит `IllegalThreadStateException`.

---
## 📊 Отличия обычного потока от демона

|                            | **Обычный (User)**                              | **Демон (Daemon)**                              |
| -------------------------- | ----------------------------------------------- | ----------------------------------------------- |
| ***JVM* ждёт завершения?** | ✅ Да                                            | ❌ Нет                                           |
| **Может быть убит *JVM***  | ❌ Нет                                           | ✅ Да (при завершении JVM)                       |
| **Пример**                 | Main, рабочие потоки                            | GC, Finalizer, фоновые таймеры                  |
| **Наследование**           | Дочерний поток наследует daemon-статус родителя | Дочерний поток наследует daemon-статус родителя |

---
## 🧠 Важный нюанс (*для собеседования*):

> _«Если поток-демон выполняет операцию записи в файл или БД, а JVM завершается — операция может быть прервана на полпути. Поэтому **не используйте демоны для критичных задач**, требующих гарантированного завершения (например, сохранение данных).»_
---
## 🎤 Идеальный ответ на собеседовании:

> _«Потоки-демоны — это фоновые потоки, которые не препятствуют завершению JVM. Они нужны для выполнения вспомогательных задач, таких как сборка мусора, мониторинг или кэширование. Создаются через `setDaemon(true)` до вызова `start()`. Важно помнить, что JVM завершится, даже если демоны еще работают, поэтому они не подходят для критических операций, где важна гарантия завершения записи данных.»_
