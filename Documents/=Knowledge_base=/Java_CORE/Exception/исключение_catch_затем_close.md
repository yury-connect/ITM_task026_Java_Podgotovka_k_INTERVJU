# Что произойдет если исключение будет выброшено из блока `catch` после чего другое исключение будет выброшено из метода `close()` при использовании `try-with-resources`?

---
# Приоритеты исключений в конструкции `try-with-resources`

Конструкция `try-with-resources` автоматически управляет закрытием ресурсов через метод `close()`, но при возникновении множественных ошибок вступает в силу жесткая иерархия приоритетов Java-рантайма.

## 🚨 Сценарий 1: Конфликт `try` против `close()` (Автоматическое подавление)

Если ошибка происходит внутри логики блока `try`, а затем еще одна ошибка возникает автоматически при попытке закрыть ресурс в методе `close()`.
```Java
try (MyResource res = new MyResource()) {
    throw new RuntimeException("Ошибка из TRY"); // 1. Основное исключение
} // 2. Автовызов res.close() -> выбрасывает "Ошибка из CLOSE"
```

- **Результат:** Исключение из блока `try` признается главным и летит вверх по стеку.
    
- **Что с ошибкой из `close()`?** Она не теряется, а автоматически прикрепляется к основному исключению как **подавленная (Suppressed Exception)**.
    
- **Как извлечь:** Через метод `throwable.getSuppressed()`.

## 💥 Сценарий 2: Конфликт `catch` против `close()` (Ловушка на собеседовании)

Если ошибка из `try` вызвала автозакрытие ресурсов, метод `close()` выбросил свою ошибку, а затем управление перешло в блок `catch`, который сгенерировал **третье, новое исключение**.

- **Результат:** Новое исключение из блока `catch` **полностью затирает (проглатывает)** всё, что было до него.
    
- **Что теряется?** Из памяти рантайма полностью стирается как первоначальная ошибка из `try`, так и уже прикрепленная к ней подавленная ошибка из метода `close()`. Основным в стеке становится исключительно эксепшн из `catch`.

## 👑 Иерархия «силы» исключений в Java (Шпаргалка)

Если в рамках одного метода происходит каскад ошибок в разных блоках, итоговым выжившим исключением в стеке станет то, которое находится **выше** в этом списке:

|**Приоритет**|**Источник исключения**|**Что происходит с предыдущими ошибками в цепочке?**|
|---|---|---|
|**1** 👑|**Блок `finally`** (ручной)|**Уничтожает всё.** Затирает ошибки из `try`, `catch` и `close()`.|
|**2** 🥈|**Блок `catch`**|**Затирает прошлое.** Уничтожает ошибку из `try` и все накопленные в ней подавленные ошибки от `close()`.|
|**3** 🥉|**Метод `close()`**|**Подавляет соседей.** Если ресурсов несколько, первая ошибка из `close()` станет главной для этой фазы, а остальные `close()` прикрепятся к ней как _Suppressed_. Но все они уступят ошибке из `try`.|
|**4** 🏅|**Блок `try`**|**Собирает мусор.** Становится главным, только если его не перекрыли `catch` или `finally`. Вбирает в себя все ошибки от `close()` как _Suppressed_.|
