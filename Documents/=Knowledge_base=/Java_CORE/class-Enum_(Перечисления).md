**Enum (Перечисления)** в Java — это полноценный класс с особым синтаксисом, представляющий фиксированный набор логически связанных констант. Это не просто список строк или чисел, а мощный инструмент с богатым функционалом.

### ⚙️ Анатомия и скрытые особенности Enum

Под капотом каждое перечисление неявно расширяет базовый класс `java.lang.Enum` и обладает жесткими правилами:

1. **Константа — это объект:** Каждая объявленная константа является **публичным**, **статическим** и **финальным** (`public static final`) экземпляром самого этого enum-класса.
    
2. **Конструктор всегда `private`:** Он может быть <u>только приватным</u> (*модификатор можно не писать, он выставится неявно*). Извне через `new` создать объект enum невозможно.
    
3. **Запрет наследования:** Enum нельзя унаследовать от другого класса (*так как множественное наследование в Java запрещено, а место родителя уже занято `java.lang.Enum`*). Сам enum автоматически является `final`, поэтому от него тоже нельзя унаследоваться.
    
4. **Интерфейсы разрешены:** Enum **может** реализовывать любые интерфейсы (`implements`).

### 🛠️ Практический пример: Поля, методы и кастомная логика

Enum позволяет связывать константы с данными и даже переопределять методы для конкретных элементов:
```Java
public enum OrderStatus {
    NEW(10) {
        @Override
        public void logStatus() { System.out.println("Создан новый заказ!"); }
    },
    IN_PROGRESS(20),
    DELIVERED(30);
	
    private final int code; // У enum могут быть обычные поля
	
    // Приватный конструктор
    OrderStatus(int code) {
        this.code = code;
    }
	
    public int getCode() { return code; }
	
    // Общий метод для всех констант
    public void logStatus() {
        System.out.println("Статус: " + this.name());
    }
}
```

### 📦 Встроенные методы Enum
Каждое перечисление автоматически получает набор полезных методов из коробки:

- `values()` — _статический_ метод, возвращает массив всех констант в порядке их объявления.
    
- `valueOf(String name)` — _статический_ метод, находит константу по её строковому имени (выбросит `IllegalArgumentException`, если имя не совпало).
    
- `name()` — возвращает точное строковое имя констант (например, `"IN_PROGRESS"`).
    
- `ordinal()` — возвращает порядковый номер константы (начиная с `0`).  
  **Зависеть от него в бизнес-логике — антипаттерн**, так как при перестановке констант местами номера изменятся.

### ⚖️ **Плюсы** и **минусы**: По сравнению со `static final int`
До появления enum разработчики использовали числовые константы (`public static final int STATUS_NEW = 1;`).

#### **Преимущества** Enum:
- **Строгая типобезопасность:** Метод, принимающий `OrderStatus`, физически не примет число `999` или константу из другого перечисления (например, дня недели). Компилятор отловит ошибку сразу.
    
- **Инкапсуляция логики:** Внутри enum можно хранить коды ошибок, переводы, правила переходов состояний, избавляя остальной код от `if-else` лапши.
    
- **Идеален для `switch`:** Прекрасно читается в блоках выбора.

#### **Ограничения** и минусы:
- **Потребление памяти:** Каждая константа — это полноценный объект в куче (`Heap`). Числовой примитив занимает в разы меньше памяти. В высоконагруженных или memory-critical системах это нужно учитывать.
    
- **Ограничение сравнений:** Объекты enum можно сравнивать только через `==`, `equals()` или `compareTo()`. Операторы порядка (`>`, `<`, `>=`, `<=`) с ними **не работают**.
