## ⚡ Динамическая загрузка классов
Динамическая загрузка — это механизм, позволяющий JVM загружать новые классы в память прямо во время выполнения программы, когда они физически не были известны на этапе компиляции. Это основа архитектур плагинов, рефлексии (Reflection API) и работы фреймворков (Spring, Hibernate).

В Java динамически загрузить класс можно двумя основными инструментами, имеющими принципиальное отличие:

### 🛠️ Инструмент 1: <br>`Class.forName("ПолноеИмяКласса")`
	
- **Что делает:** Загружает байт-код класса, проводит связывание и **обязательно запускает инициализацию** (выполняет статические блоки кода и инициализирует статические поля).
    
- **Сценарий:** Классический пример — старая загрузка драйверов СУБД JDBC (`Class.forName("org.postgresql.Driver")`), когда при вызове метода драйвер должен был сам статически зарегистрировать себя в системе.

### 🛠️ Инструмент 2: <br>`ClassLoader.loadClass("ПолноеИмяКласса")`
	
- **Что делает:** Напрямую задействует подсистему загрузчиков. Он только **загружает** класс в память, но **НЕ инициализирует его**. Статические блоки выполнены не будут, пока к классу не обратятся напрямую (например, через создание экземпляра).

## 🚨 8. Главный чек-лист распространенных ошибок JVM

| **Ошибка / Исключение**      | **Тип**           | **Почему возникает на самом деле**                                                                                                                                                                                                                                |
| ---------------------------- | ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`ClassNotFoundException`** | Checked Exception | Запрос на динамическую загрузку класса (`Class.forName()`, `loadClass()`) выполнен по строковому имени, но класс с таким именем **физически отсутствует** в доступных путях поиска загрузчиков.                                                                   |
| **`NoClassDefFoundError`**   | Error             | Компиляция прошла успешно, файл `.class` существовал. Однако во время выполнения JVM попыталась обратиться к этому классу, но загрузчик классов **не смог найти его файл в текущем classpath** (например, jar-архив удалили или не донесли на сервер при деплое). |
| **`OutOfMemoryError`**       | Error             | Заполнена физическая память Области Кучи (Heap) или Области метода (Metaspace). Новые объекты не могут быть созданы, так как свободного места нет, а Сборщик мусора (GC) не может высвободить память (все объекты удерживаются сильными ссылками).                |
| **`StackOverflowError`**     | Error             | В области стека конкретного потока закончилось свободное место для создания нового фрейма. Самая частая причина — некорректная, бесконечная рекурсия или чрезмерно глубокий вызов методов.                                                                        |
