# Двоичное (Бинарное) дерево / Binary Tree

**Двоичное (бинарное) дерево** — это иерархическая структура данных, в которой каждый узел имеет **не более двух потомков** (обычно называемых _левым_ (`left`) и _правым_ (`right`) поддеревьями/детьми).

В отличие от линейных структур (массивов, списков), где каждый элемент связан только со следующим, бинарное дерево ветвится, предоставляя основу для эффективных алгоритмов поиска, сортировки и иерархического представления данных.

---
## 🏗️ 1. Анатомия и базовый код узла

Физически в оперативной памяти бинарное дерево представляет собой набор объектов-узлов, связанных между собой через указатели (ссылки).

Классическая реализация узла бинарного дерева на Java выглядит так:
```Java
public class TreeNode {
    int value;          // Хранимые данные (может быть любым объектом/типом)
    TreeNode left;      // Ссылка на левого потомка (null, если потомка нет)
    TreeNode right;     // Ссылка на правого потомка (null, если потомка нет)

    public TreeNode(int value) {
        this.value = value;
        this.left = null;
        this.right = null;
    }
}
```

## 📊 2. Классификация бинарных деревьев

В зависимости от распределения узлов и заполненности уровней, бинарные деревья делятся на несколько важных типов. Эта классификация критична для оценки эффективности алгоритмов:

- **Строгое / Полное (Full / Strict Binary Tree):** Дерево, у которого каждый узел имеет либо ровно **0**, либо ровно **2** потомков. Узлов с одним ребенком в таком дереве быть не может.
    
- **Совершенное / Идеальное (Perfect Binary Tree):** Дерево, в котором все внутренние узлы имеют ровно двух детей, а все листья находятся строго **на одном (самом нижнем) уровне**.
    
- **Завершенное / Почти полное (Complete Binary Tree):** Дерево, в котором все уровни полностью заполнены, за исключением, возможно, последнего. При этом на последнем уровне все узлы должны быть сгруппированы **максимально слева**.
    
    - _Где применяется:_ Идеально для реализации структуры **Бинарная Куча (Binary Heap)**, используемой в очередях с приоритетом и алгоритме сортировки _Heapsort_.
        
- **Вырожденное (Degenerate / Pathological Tree):** Дерево, у которого каждый узел имеет ровно одного потомка (кроме последнего листа). Фактически такое дерево превращается в обычный **односвязный список**, а эффективность древесных операций падает с $O(\log N)$ до худших $O(N)$.
    
- **Сбалансированное по высоте (Balanced Binary Tree):** Дерево, у которого для любого узла высота его левого и правого поддеревьев отличается **не более чем на 1**.
    
    - _Зачем нужно:_ Гарантирует, что дерево не выродится в список, и все операции поиска/вставки будут выполняться за логарифмическое время $O(\log N)$.

## 🏃 3. Способы обхода бинарного дерева (Tree Traversal)

Обход — это процесс последовательного посещения всех узлов дерева ровно по одному разу. Поскольку структура разветвленная, существует два фундаментальных подхода: **В ширину (BFS)** и **В глубину (DFS)**.

### ── А. Обход в ширину (Breadth-First Search / BFS)
Дерево обходится горизонтально, послойно, уровень за уровнем сверху вниз (сначала корень, потом все его дети, затем все внуки).

- **Как реализуется:** Итеративно с помощью структуры данных **Очередь (Queue)**.
    
- **Название алгоритма:** Обход по уровням (_Level-order traversal_).
    

### ── Б. Обходы в глубину (Depth-First Search / DFS)
Мы спускаемся по одной ветке до самого глубокого листа, прежде чем перейти к соседней. Существует 3 канонических порядка обхода (реализуются через **Рекурсию** или явный **Стек**):
```
        [Корень]
        /      \
   [Лево]      [Право]
```

1. **Прямой обход (Pre-order): `Корень ──► Лево ──► Право`**
    
    - _Применение:_ Используется для создания точной копии дерева или сериализации структуры в строку/файл.
        
2. **Центрированный / Симметричный (In-order): `Лево ──► Корень ──► Право`**
    
    - _Применение:_ Если применить этот обход к _Двоичному дереву поиска (BST)_, мы гарантированно обойдем все элементы в **строго отсортированном по возрастанию порядке**.
        
3. **Обратный обход (Post-order): `Лево ──► Право ──► Корень`**
    
    - _Применение:_ Используется при удалении дерева из памяти (так как родительский узел можно безопасно удалить только после того, как были удалены оба его потомка) или для подсчета объема файлов в иерархии папок.

## 🎯 4. Ключевые свойства и математические закономерности

Знание этих свойств помогает оценивать затраты памяти при проектировании древесных структур:

- Максимальное количество узлов на любом уровне $i$ (где корень находится на уровне $0$) равно:
    $$N_{\max} = 2^i$$
    
- Максимальное общее количество узлов в идеальном бинарном дереве высоты $h$ рассчитывается по формуле:
    $$N = 2^{h+1} - 1$$
    
- Минимально возможная высота бинарного дерева с общим числом узлов $N$ составляет:
    $$h_{\min} = \lfloor\log_2 N\rfloor$$
    
- Связь между количеством листьев ($L$) и количеством внутренних узлов с двумя детьми ($N_2$) в любом бинарном дереве всегда подчиняется строгому равенству:
    $$L = N_2 + 1$$

## ⚖️ 5. Преимущества и недостатки

**Плюсы:**

- **Эффективное масштабирование:** В сбалансированном виде обеспечивает логарифмическую сложность $O(\log N)$ для базовых операций, что критически быстрее линейных массивов при больших объемах данных.
    
- **Гибкость структуры:** Позволяет легко расширять структуру данных без необходимости выделения единого непрерывного блока памяти в куче (в отличие от статических массивов).
    
- **Идеальное отражение дихотомии:** Подходит для задач, связанных с бинарным выбором («да/нет», «меньше/больше», сжатие данных по Хаффману).
    

**Минусы:**

- **Оверхед по памяти:** Каждый элемент требует создания объекта и хранения двух (а иногда трех, включая ссылку на `parent`) физических указателей-ссылок в памяти.
    
- **Риск деградации:** Без принудительных алгоритмов самобалансировки (как в AVL или Красно-черных деревьях) велика вероятность превращения дерева в неэффективный односвязный список.
