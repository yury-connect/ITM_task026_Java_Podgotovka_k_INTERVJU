## 🚨 Важные особенности работы `TreeMap`, о которых спросят на интервью

1. **Скорость работы $O(\log N)$:** Из-за древесной структуры операции поиска (`get`), вставки (`put`) и удаления (`remove`) работают медленнее, чем в `HashMap` (где они занимают $O(1)$). Здесь рантайму нужно каждый раз спускаться по веткам дерева от корня к листьям.
    
2. **Игнорирование `equals()`:** `TreeMap` **вообще не использует** метод `equals()` для проверки уникальности ключей. Она считает два ключа одинаковыми (дубликатами) только если метод `compareTo()` (или `compare()`) вернул **`0`**.
    
3. **Запрет на `null`:** В `TreeMap` **нельзя** положить `null` в качестве ключа. Так как при добавлении мапа обязана вызвать метод сравнения (`null.compareTo(другой_ключ)`), программа мгновенно упадет с `NullPointerException`.
    
4. **Навигационные методы:** Благодаря сортировке, у `TreeMap` есть крутые уникальные методы, которых нет у `HashMap`:
    
    - `firstKey()` / `lastKey()` — получить самый маленький / самый большой ключ.
        
    - `subMap(fromKey, toKey)` — получить кусок мапы в диапазоне между двумя ключами.
        
    - `headMap(toKey)` / `tailMap(fromKey)` — получить «голову» или «хвост» мапы от указанного ключа
