# Сколько памяти занимает **boolean**?

Согласно спецификации **JVMS (глава 2.3.4)**, для одиночных переменных типа `boolean` компилятор Java (`javac`) использует тип **`int`** (*а значит, выделяется **4 байта***) только в тех случаях, когда над ними выполняются выражения или они лежат в стеке локальных переменных.

Но если `boolean` — это **поле класса (instance variable)**, то в HotSpot JVM оно занимает **1 байт**, а не 4!

Давай структурируем точные размеры `boolean` в стандартной HotSpot JVM с примерами:

---
### 1. Одиночная локальная переменная внутри метода $\rightarrow$ <br>$\rightarrow$ 4 байта (`int`)
На уровне байткода для типа `boolean` нет собственных инструкций арифметики или логики. JVM маппит их на команды `int` (где `0` — это `false`, а `1` — это `true`).

**Пример:**
```Java
public void myMethod() {
    boolean flag = true; // В стеке потока под эту переменную выделится 4 байта (как под int)
}
```

### 2. Поле внутри класса (Instance variable) $\rightarrow$ 1 байт
Если переменная объявлена как поле класса, JVM оптимизирует память и выделяет под неё ровно **1 байт**. Однако из-за механизма **выравнивания памяти (Memory Alignment)** до ближайшего числа, кратного 8 байтам, этот 1 байт может визуально «раздуть» объект, если рядом нет других мелких переменных.

**Пример:**
```Java
public class User {
    boolean isActive; // Занимает 1 байт памяти внутри объекта в куче
}
```

### 3. Элемент массива `boolean[]` $\rightarrow$ 1 байт (*в HotSpot)*
Как правильно замечено на картинке, элементы массива в HotSpot кодируются байткод-инструкциями для работы с массивами типа `byte` (`baload` / `bastore`). Поэтому каждый элемент занимает ровно **1 байт**.

### 💡 Как сэкономить память, если нужен ровно 1 бит?
Если у тебя высоконагруженное приложение и тебе нужно хранить миллионы флагов `true`/`false`, обычный массив `boolean[]` потратит в 8 раз больше памяти, чем мог бы. В стандартной Java для «упаковки» данных по 1 биту на элемент используют встроенный класс **`java.util.BitSet`**.

Под капотом `BitSet` использует массив чисел `long[]`. Одно число `long` занимает 64 бита, и внутри него с помощью побитовых сдвигов (`&`, `|`) упаковываются 64 независимых логических флага.

**Сравнение в коде:**
```Java
// Потратит 10 000 000 байт (около 9.5 МБ)
boolean[] standardArray = new boolean[10_000_000]; 

// Потратит ровно 10 000 000 бит (около 1.19 МБ) — экономия в 8 раз!
BitSet efficientSet = new BitSet(10_000_000); 
efficientSet.set(500, true);
boolean value = efficientSet.get(500);
```

### Резюме для ответа на собеседовании:
1. **Локальная переменная:** 4 байта 
	   (*так как JVM оперирует ею через инструкции для `int`*).
    
2. **Поле класса:** 1 байт (*плюс возможный паддинг выравнивания*).
    
3. **В массиве:** 1 байт на элемент в стандартных JVM из-за архитектуры процессоров. 1 бит на элемент бывает только в специфических Embedded-системах. Для ручной оптимизации до 1 бита в Java используют `BitSet`.
