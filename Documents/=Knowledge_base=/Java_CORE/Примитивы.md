26. Какие примитивные типы данных есть в Java?

В Java есть 8 примитивных типов данных, они делятся на:

> 1. **Целочисленные**> 
> - **byte** (8 бит, от -128 до 127)>     
> - **short** (16 бит, от -32 768 до 32 767)>     
> - **int** (32 бита, от -2^31 до 2^31-1)>     
> - **long** (64 бита, от -2^63 до 2^63-1)>     

> 2. **Вещественные** (числа с плавающей запятой)> 
> - **float** (32 бита, 7 знаков после запятой)
> - **double** (64 бита, 15 знаков после запятой)

> 3. **Символьный**> 
> - **char** (16 бит, хранит один символ в формате UTF-16)

> 4. **Логический**> 
> - **boolean** (принимает только true или false)

**Важно**:
- Примитивные типы данных не являются объектами и хранятся в стеке.
- String не является примитивным типом, это ссылочный _(объектный)_ тип

# Инициализация по умолчанию:
В Java значения по умолчанию получают **только поля класса** (как статические, так и нестатические).

### 📊 Таблица значений по умолчанию для типов данных

|**Тип данных**|**Значение по умолчанию**|**Примечание / Суффикс**|
|---|---|---|
|**`byte`**|`0`|Специального суффикса нет. Можно привести явно: `byte b = (byte) 0;`|
|**`short`**|`0`|Специального суффикса нет. Можно привести явно: `short s = (short) 10;`|
|**`int`**|`0`|Базовый целочисленный тип.|
|**`long`**|`0L`|Использует суффикс `L` (или `l`).|
|**`float`**|`0.0f`|Использует суффикс `f` (или `F`).|
|**`double`**|`0.0d`|Использует суффикс `d` (или `D`), но он необязателен.|
|**`char`**|`'\u0000'`|Нулевой символ (символ конца строки / `null`-символ).|
|**`boolean`**|`false`|Логический тип всегда инициализируется ложью.|
|**Объекты**|`null`|Любые ссылочные типы данных (включая `String` и массивы).|

### ⚠️ Важные правила инициализации, которые нужно помнить:

- **Локальные переменные (внутри методов):** Не имеют значений по умолчанию. Они вообще не инициализируются автоматически JVM. Если попытаться использовать локальную переменную до того, как ей явно присвоили значение, программа выдаст **ошибку компиляции** (`Variable might not have been initialized`).
    
- **Момент инициализации полей класса:**
    
    - **`static` (статические) поля** — инициализируются один раз в момент, когда сам класс загружается виртуальной машиной (JVM) в память (в область Metaspace).
        
    - **Нестатические поля (поля объекта)** — инициализируются каждый раз заново в куче (Heap), непосредственно при создании нового экземпляра (объекта) этого класса.
