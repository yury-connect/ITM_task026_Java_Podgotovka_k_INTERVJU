### **Вывод: какие пулы есть в Java?**

| Класс       | Пулл (кэш)       | Диапазон                  | Можно расширить?                                 |
| ----------- | ---------------- | ------------------------- | ------------------------------------------------ |
| `Integer`   | `IntegerCache`   | -128 до 127               | Да (`-Djava.lang.Integer.IntegerCache.high=500`) |
| `Long`      | `LongCache`      | -128 до 127               | Нет                                              |
| `Short`     | `ShortCache`     | -128 до 127               | Нет                                              |
| `Byte`      | `ByteCache`      | -128 до 127 (весь `byte`) | Нет                                              |
| `Character` | `CharacterCache` | 0 до 127 (ASCII)          | Нет                                              |
| `Boolean`   | `TRUE`, `FALSE`  | только `true` и `false`   | Нет                                              |
| `String`    | String Pool      | Все литералы и `intern()` | Нет (но можно добавлять через `intern()`)        |

### **Максимальный диапазон пула `Integer` (IntegerCache) в Java**

В Java пул `Integer` (кэш) по умолчанию хранит объекты для значений **от -128 до 127**, но его можно расширить с помощью JVM-параметра:
```bash
-Djava.lang.Integer.IntegerCache.high=<максимальное_значение>
```

### **1. До какого значения можно расширить?**

- **Теоретически** – до `Integer.MAX_VALUE - 128` (т.е. до `2_147_483_519`), но на практике **разумный предел гораздо меньше**.
    
- **Рекомендуемый максимум** – **не более нескольких тысяч** (например, `-Djava.lang.Integer.IntegerCache.high=10000`), иначе:
    
    - **Увеличивается потребление памяти** (каждое значение кэша — это отдельный объект `Integer`).
        
    - **Замедляется старт JVM** (инициализация пула происходит при запуске).
        

### **2. Как проверить текущий размер пула?**
```java
public class IntegerCacheTest {
    public static void main(String[] args) {
        // Выводим верхнюю границу кэша (Java 9+)
        System.out.println("Integer cache high: " + sun.misc.VM.getSavedProperty("java.lang.Integer.IntegerCache.high"));
        
        // Проверяем, берётся ли значение из пула
        Integer a = 1000;
        Integer b = 1000;
        System.out.println(a == b); // true, если 1000 входит в пул
    }
}
```

### **3. Пример расширения пула**
Запуск программы с увеличенным пулом:
```bash
java -Djava.lang.Integer.IntegerCache.high=5000 MyApp
```

### **4. Ограничения**
- **Не влияет на `new Integer()`** – всегда создаёт новый объект.    
- **Только для `Integer.valueOf()` и автоупаковки** (`Integer i = 100`).    
- **Не работает для `Long`, `Short` и других** – только `Integer`.    

### **5. Зачем расширять пул?**
- **Оптимизация памяти**, если в приложении часто используются одни и те же числа (например, ID в определённом диапазоне).    
- **Ускорение сравнения через `==`** (но лучше всё равно использовать `equals()`!).
