# отличие: `List<?>` против `List<T>`

---
### Главное отличие: `List<?>` против `List<T>`

Ключевая разница заключается в том, **где** объявлен тип и насколько жестко мы его фиксируем.

## 1. `List<T>` (*Конкретный generic-параметр*)
Буква `T` — это **переменная типа** (Type Variable). Она обязана быть объявлена на уровне класса или метода. Когда мы используем `T`, мы говорим: _«Мы не знаем заранее, что это за тип, но он **один и тот же** во всех местах, где используется эта буква `T`»_.

- **Пример применения:**
```Java
// T объявлен на уровне метода. 
// Мы гарантируем: какой тип придет в аргумент, такой же тип и вернется!
public <T> T getFirst(List<T> list) {
	return list.get(0); 
}
```
   
Если мы передали `List<String>`, компилятор жестко знает, что `T` — это `String`, и метод вернет именно `String`. В `List<T>` **можно добавлять элементы** типа `T`.    

## 2. `List<?>` (*Unbounded Wildcard — неизвестный тип*)
Символ `?` — это маска подстановки, означающая **абсолютно неизвестный тип**. Она используется сама по себе, ее не нужно нигде предварительно объявлять. Мы говорим: _«Здесь список какого-то типа, нам вообще всё равно какого»_.

- **Пример применения:**
```Java
// Метод просто печатает размер. Нам плевать, строки там или числа.
public void printSize(List<?> list) {
	System.out.println(list.size());
}
```

- **Важное ограничение:** Из-за того, что тип полностью неизвестен, `List<?>` ведет себя точно так же, как `List<? extends Object>`. Это значит, что **в него нельзя ничего добавить (`.add()`)**, кроме `null`. Компилятор блокирует запись ради безопасности.
