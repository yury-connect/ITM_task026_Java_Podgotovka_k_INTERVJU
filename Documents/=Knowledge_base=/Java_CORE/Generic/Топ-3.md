### **Топ-3** каверзных вопросов по этой теме для самопроверки

Сделай паузу и попробуй мысленно ответить на них перед тем, как читать спойлеры.

---
#### Вопрос 1. <br>Скомпилируется ли этот код и почему?
```Java
List<Number> numList = new ArrayList<Integer>(); // Строка 1
List<?> wildList = new ArrayList<Integer>();     // Строка 2
```

- **Строка 1 — НЕ скомпилируется.** Дженерики в Java являются **инвариантными**. Несмотря на то, что `Integer` наследуется от `Number`, класс `List<Integer>` **НЕ** является наследником `List<Number>`. Если бы Spring или Java разрешали такое, мы могли бы через ссылку `List<Number>` положить в список интеджеров число `3.14 (Double)`, сломав рантайм.
    
- **Строка 2 — Скомпилируется отлично.** Маска `?` означает "любой тип", поэтому `List<?>` является супертипом (родителем) для абсолютно любого параметризованного списка.    

---
#### Вопрос 2. <br>Какая разница между `List<Object>` и `List<?>`?
- В `List<Object>` **можно добавлять любые объекты** (`String`, `Integer` и т.д.). Но в переменную этого типа можно присвоить **только** `new ArrayList<Object>()`. Передать туда `new ArrayList<String>()` нельзя (из-за инвариантности, см. Вопрос 1).
    
- В `List<?>` **нельзя ничего добавить** (кроме `null`). Но в переменную этого типа можно присвоить абсолютно **любой список в Java** (`List<String>`, `List<Integer>`, `List<MyDTO>`).    

---
#### Вопрос 3. <br>Что вернет метод `.get(0)` <br>для коллекций `List<? extends Number>` и `List<? super Number>`? <br>В переменные какого типа можно сохранить результат?

- Для `List<? extends Number>` метод `.get()` вернет элемент типа **`Number`**. Мы точно знаем верхнюю границу, поэтому компилятор разрешает сохранить результат в `Number num = list.get(0);`.
    
- Для `List<? super Number>` метод `.get()` вернет только **`Object`**. Мы знаем лишь нижнюю границу (это `Number` или его родители выше по древу). Родителем может оказаться как `Number`, так и `Object`. Единственное, в чем компилятор уверен на 100% — это то, что любой объект в Java является объектом. Поэтому: `Object obj = list.get(0);`.
