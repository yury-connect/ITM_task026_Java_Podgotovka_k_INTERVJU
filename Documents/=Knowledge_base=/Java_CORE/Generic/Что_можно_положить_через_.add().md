Что можно положить через **`.add()`** *сюда*: ?
### 1. **`List<? extends Number> numbers = new ArrayList<>()`**
### 2. **`List<? super Number> numbers = new ArrayList<Object>();`**

---
Это классический вопрос на понимание концепции **PECS (*Producer Extends, Consumer Super*)** из темы Java Generics Wildcards (*масок подстановки*). Ответ на него часто шокирует тех, кто только начинает разбираться с дженериками.

Давай разберем каждую строчку и обоснуем поведение компилятора.

### Вопрос 1: <br>**`List<? extends Number> numbers = new ArrayList<>()`**

**Что можно положить через `.add()`?**
> **Ответ:** Сюда **нельзя положить абсолютно ничего** (кроме `null`).

#### Обоснование:
Запись `? extends Number` (Upper Bound Wildcard) означает: _«Здесь лежит список какого-то конкретного типа, который является наследником `Number` (например, это может быть `List<Integer>`, `List<Double>` или `List<Long>`), но какого именно — мы не знаем»_.

Представь, что компилятор разрешил бы тебе добавить туда число:
```Java
List<? extends Number> numbers = new ArrayList<Integer>(); // Справа легитимный List<Integer>
numbers.add(3.14); // Ошибка! Если бы это сработало, мы бы добавили Double в список Integer!
```

Так как компилятор не имеет ни малейшего понятия, список _каких именно_ чисел ему подсунули справа от знака равно, он **блокирует любые попытки записи**, чтобы защитить программу от нарушения типов (Type Safety).

- **Для чего нужен такой список?** Исключительно для чтения! Мы можем безопасно читать из него элементы, и они гарантированно будут типа `Number` или его наследниками (*это **Producer** из правила PECS*).

---
### Вопрос 2: <br>**`List<? super Number> numbers = new ArrayList<Object>();`**

**Что можно положить через `.add()`?**
> **Ответ:** Сюда можно положить **сам `Number`**, 
> а также **любых его наследников** (`Integer`, `Double`, `Long`, `Float`, `Byte`, `Short`).

#### Обоснование:
Запись `? super Number` (Lower Bound Wildcard) означает: _«Здесь лежит список типа `Number` или любого его родителя (например, `List<Number>` или `List<Object>`)»_.

Компилятор рассуждает так: если в самый «узкий» возможный список (`List<Number>`) можно положить любой объект, который является `Number` (или его подтипом), то в любой более широкий список родительских классов (например, в `List<Object>`) эти элементы тоже гарантированно и безопасно поместятся.

Поэтому следующий код отработает идеально:
```Java
List<? super Number> numbers = new ArrayList<Object>();
numbers.add(Integer.valueOf(10)); // OK: Integer — это Number
numbers.add(Double.valueOf(5.5)); // OK: Double — это Number
```

- **Для чего нужен такой список?** В первую очередь для записи! Мы можем безопасно поставлять туда данные (это **Consumer** из правила PECS). Читать из него сложнее — при чтении мы получим только самый базовый тип `Object`, потому что компилятор не знает, насколько «высоко» по иерархии классов поднялся этот список.    

---
### Резюме (Правило PECS)
- `? extends T` — список работает как **Producer** (производитель). Из него можно только **брать** данные типа `T`. Запись в него закрыта.
    
- `? super T` — список работает как **Consumer** (потребитель). В него можно **класть** данные типа `T` и его наследников. При чтении возвращается только `Object`.
