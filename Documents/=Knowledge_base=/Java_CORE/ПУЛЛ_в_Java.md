В Java пулы используются для оптимизации производительности и экономии памяти за счет повторного использования тяжелых или часто создаваемых объектов.  
Ниже приведена конкретная и структурированная информация по всем основным пулам в экосистеме Java.  

---
## 1. Пул **строк** (*String Pool*)
Хранит уникальные строковые литералы **в куче** (*Heap*), предотвращая дублирование одинаковых строк в памяти.
	
- **Диапазон / Вместимость:** Ограничен только размером доступной кучи.  
  Технически пул реализован как хэш-таблица (`StringTable`). По умолчанию ее размер (`-XX:StringTableSize`) составляет **65 536** элементов (*начиная с Java 11*).
    
- **Как повлиять (*из кода*):** Метод `string.intern()` принудительно отправляет строку в пул или возвращает ссылку, если она там уже есть.
    
- **Сжатие / Очистка:** * **Очистка:** Строки из пула автомат удаляются **GC**, если на них больше <u>нет сильных ссылок</u> в приложении (*начиная с Java 7*).
    
    - **Сжатие:** Начиная с Java 9, включена технология **Compact Strings** (сжатие на уровне JVM): если строка содержит только символы ISO-8859-1/Latin-1, под каждый символ выделяется 1 байт вместо 2 (массив `byte[]` вместо `char[]`), что косвенно сжимает и сам пул.
    
- **Тюнинг JVM:** Можно увеличить размер таблицы для уменьшения коллизий: `-XX:StringTableSize=100003`.

---
## 2. Пулы кеширования **примитивов** <br>(*Primitive Wrappers Cache*)
JVM автоматически кеширует мелкие значения классов-оберток при операциях автоупаковки (*autoboxing*), чтобы не создавать новые объекты в куче.

### Кэш Integer (Integer Cache)
- **Диапазон по умолчанию:** От **`-128`** до **`127`**.
    
- **Как повлиять:** Верхнюю границу можно расширить флагом JVM: `-XX:AutoBoxCacheMax=<new_size>`. Нижнюю границу изменить нельзя. Сжать или принудительно очистить этот пул в рантайме невозможно — он создается один раз при старте JVM.

### Кэши других примитивов
- **`Byte`**, **`Short`**, **`Long`**, **`Character`**: Имеют жестко зафиксированный диапазон от **`-128`** до **`127`** (для `Character` от `0` до `127`). Изменить или настроить эти диапазоны через флаги JVM нельзя.
    
- **`Boolean`**: Кеширует два единственных значения: `Boolean.TRUE` и `Boolean.FALSE`.

## 3. Пул **потоков** (*Thread Pool*)
Управляет переиспользованием потоков ОС через фреймворк `ExecutorService`, исключая накладные расходы на постоянное создание и уничтожение потоков.
	
- **Диапазон / Вместимость:** Задается разработчиком при создании пула. Основные параметры:    
    - `corePoolSize` — минимальное количество постоянно удерживаемых потоков. 
    - `maximumPoolSize` — максимально допустимое количество потоков.
    - `keepAliveTime` — время, через которое лишние (простаивающие) потоки будут уничтожены.
    
- **Сжатие / Изменение размера:** Большинство реализаций (например, `ThreadPoolExecutor`) позволяют динамически менять размеры пула прямо в рантайме с помощью методов `.setCorePoolSize(int)` и `.setMaximumPoolSize(int)`.
    
- **Очистка / Закрытие:** * Метод `.shutdown()` плавно закрывает пул, запрещая прием новых задач, но дожидаясь выполнения старых.
    - Метод `.shutdownNow()` пытается немедленно прервать текущие задачи и очищает очередь ожидания.
        
    - Метод `.allowCoreThreadTimeOut(true)` позволяет уничтожать даже core-потоки, если они простаивают, сжимая пул до 0.

---
## 4. Пул **соединений с базой данных** (*Connection Pool*)
Сторонняя библиотека (например, **HikariCP**, **Tomcat DBCP**), которая удерживает открытыми сетевые соединения с БД.
	
- **Диапазон:** Задается в конфигурации (параметры `minimumIdle` и `maximumPoolSize`). По умолчанию в HikariCP макс. размер равен **10**.
    
- **Как повлиять / Сжать:** Пулы динамически сжимаются (закрывают лишние коннекты) до уровня `minimumIdle`, если они не используются дольше таймаута `idleTimeout`.
    
- **Очистка:** Полная очистка и закрытие пула происходят при вызове метода `.close()`. Сбросить «битые» соединения можно вызовом метода фонового тестирования соединений (`keepaliveTime`).

---
## 5. Пул **констант** (*Runtime Constant Pool*)
Системная область памяти внутри **Metaspace** (выделяется из native memory ОС) для каждого отдельного загруженного класса.
	
- **Диапазон:** Ограничен физической памятью сервера или флагом `-XX:MaxMetaspaceSize`.
    
- **Что содержит:** Ссылки на методы, имена классов, числовые литералы и индексы, необходимые для работы байткода данного класса.
    
- **Очистка:** Этот пул невозможно очистить или сжать вручную. Он полностью удаляется из памяти только тогда, когда сборщик мусора выгружает из Metaspace сам класс (`Class`), который его породил. Это происходит, если загрузчик класса (`ClassLoader`) становится недоступен для GC Roots.
