# Что такое сигнатура метода?

**Сигнатура метода** – это его `имя` + `список параметров` (_включая их типы и порядок_).

#### **НЕ входит** в сигнатуру:
	
> - **Возвращаемый тип**.
> - **Бросаемые исключения**.

---
**Сигнатура метода** — это его уникальный «идентификатор» для компилятора Java в пределах одного класса. Именно по сигнатуре компилятор понимает, какой конкретно метод нужно вызвать.

Ниже приведена четкая структура того, что входит и что не входит в это понятие, а также как сигнатура соотносится с полным контрактом метода.

### 1. Что **входит** в сигнатуру метода
Сигнатура состоит всего из **двух** элементов:

- **Имя метода.**    
- **Список параметров** 
	  (включая их точные типы, количество и строгий порядок следования).    

> **Пример сигнатуры:** `registerUser(String, int, boolean)`

### 2. Что **НЕ входит** в сигнатуру метода
Эти элементы могут быть частью объявления метода, но компилятор **не учитывает** их при определении уникальности (например, при перегрузке):

- **Возвращаемый тип данных**.    
- **Бросаемые исключения** (блок `throws`).    
- Модификаторы доступа (`public`, `private` и т.д.).    
- Ключевые слова `static`, `final`, `synchronized`.    
- Имена самих переменных в параметрах 
	  (компилятору важны только их _типы_, а не то, как ты их назвал).

### 3. Важное расширение: Контракт метода
Если объединить всё вместе, мы получаем полную анатомию метода, которая в ООП называется контрактом:

$$\text{Контракт метода} = \text{Сигнатура} + \text{Возвращаемый тип} + \text{Исключения}$$

### 4. Как это работает при переопределении (`@Override`)
Когда класс-наследник переопределяет метод родителя, вступают в силу жесткие правила компиляции:

1. **Сигнатура метода обязана оставаться строго неизменной**. Ты не можешь изменить имя или типы параметров.
    
2. **Возвращаемый тип данных может сужаться (Ковариантность):** Наследнику разрешено возвращать более специфичный (дочерний) тип вместо базового.
    
3. **Возвращаемый тип данных НЕ может расширяться (Контравариантность):** Наследнику строго запрещено возвращать более общий (родительский) тип.
