## 🧱 Нативный стек (Native Stack) — кратко

---
## 🎯 Что такое нативный стек
	
**Нативный стек** — это область памяти, где хранятся **вызовы нативных (native) методов**, написанных на языках **C/C++** и вызванных из Java через JNI (Java Native Interface).

**Простыми словами:** Если Java-код вызывает метод, написанный на C (например, `System.currentTimeMillis()` или работу с файлами/сетью), то информация об этом вызове сохраняется в **нативном стеке**.

---
## 🧠 Аналогия для запоминания

Представьте, что ваша Java-программа — это **офис**:
	
- **Java-стек** — это документы, которые готовят сотрудники (Java-методы).
    
- **Нативный стек** — это вызов курьера на доставку (нативный метод). Курьер работает по своим правилам (на языке C/С++), и его действия фиксируются отдельно в журнале (нативном стеке).

---
## 📊 Отличия *Java*-стека от нативного стека

| Характеристика         | Java Stack (JVM Stack)                      | Native Stack                          |
| ---------------------- | ------------------------------------------- | ------------------------------------- |
| **Что хранит**         | Вызовы **Java-методов**                     | Вызовы **нативных (C/C++) методов**   |
| **Где выделяется**     | В **Heap** (куча), но как отдельная область | В **нативной памяти** (Native Memory) |
| **Размер**             | Обычно фиксированный (например, -Xss)       | Зависит от ОС, обычно меньше          |
| **Сборка мусора**      | Участвует (GC)                              | Не участвует                          |
| **Кто управляет**      | JVM                                         | Операционная система                  |
| **StackOverflowError** | ✅ Да                                        | ✅ Да (но в C, а не в Java)            |

## 🔄 Общая ли она для всех потоков?

### ❌ НЕТ! Нативный стек — **НЕ ОБЩИЙ** для всех потоков.

|Стек|Для кого|Общий?|
|---|---|---|
|**Java Stack**|**Каждый поток** имеет свой собственный Java-стек|❌ Нет|
|**Native Stack**|**Каждый поток** имеет свой собственный нативный стек|❌ Нет|

**Аналогия:** У каждого сотрудника (потока) есть свой личный журнал (стек). Один сотрудник не видит записи другого. То же самое и с нативным стеком.

**Почему?** Потоки в Java — это отдельные единицы выполнения. Когда поток выполняет нативный метод, информация об этом должна храниться отдельно от других потоков, чтобы не было путаницы.

## 💥 Что происходит, если нативный стек переполняется?
	
- В **Java** вы получите **`StackOverflowError`**.
    
- В **C/C++** это может привести к **Segmentation Fault (SIGSEGV)** и **падению всей JVM**.

**Пример:** Если нативный метод вызывает себя бесконечно (рекурсия), нативный стек переполнится, и приложение упадёт.

---
## ✅ Итог одной фразой

> **Нативный стек** — это личная область памяти каждого потока, где хранятся вызовы **C/C++ методов**, вызванных из Java через JNI.  
> **Она не общая для потоков** — у каждого потока свой нативный стек.  
> Если переполняется — можно получить **StackOverflowError** или падение JVM.
