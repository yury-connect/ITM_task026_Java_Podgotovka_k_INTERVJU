# Поколения Garbage Collection (GS)
Вот **таблица по основным JVM-GC**, с годом выхода / версией Java / плюсами/минусами и кратким описанием:

| Год/версия выхода                   | Название GC                                                                                                                                                                                                                                                         | Версия Java                    | + / − (коротко) / описание                                  |
| :---------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | :----------------------------- | :---------------------------------------------------------- |
| **~1996 /<br> JDK 1.0**             | [**Serial** GC](Serial_GC.md)                                                                                                                                                                                                           | 1.x                            | + прост, − STW паузы, одномпоточный; базовый GC             |
| **~1996 /<br> JDK 1.x**             | [**Parallel** GC](Parallel_GC.md)                                                                                                                                                                                                       | 1.x                            | + throughput, − STW на весь heap; многопоточный сбор        |
| **~2004 /<br> JDK 1.4.1**           | [**CMS** (Concurrent Mark-Sweep)](CMS_(Concurrent_Mark-Sweep).md)                                                                                                                                                                       | 1.4.1; депр. в 9; removed в 14 | + низкие паузы, − фрагментация; concurrent marking          |
| **~2012 /<br> JDK 7u4 (эксп.)**     | [**G1**](GarbageFirst_(G1)_GC_(3).md) GC (**[1](GarbageFirst_(G1)_GC_(1).md)/ [2](GarbageFirst_(G1)_GC_(2).md)/ [3](GarbageFirst_(G1)_GC_(3).md)**) | 7 (эксп.), default с 9         | + сбалансирован, регионы, предсказуемые паузы; − сложнее    |
| **~2018 /<br> JDK 11 (эксп.)**      | [**ZGC**](ZGC_GC.md)                                                                                                                                                                                                                    | 11 (эксп.), stable позже       | + сверхнизкие паузы, масштабируем; − CPU/memory overhead    |
| **~2018–2019 /<br> JDK 12 (эксп.)** | [**Shenandoah GC**](Shenandoah_GC.md)                                                                                                                                                                                                   | 12 (эксп.)                     | + низкие паузы, concurrent compaction; − высокие ресурсы    |
| **~2019 /<br> JDK 11+**             | [**Epsilon** GC](Epsilon_GC.md)                                                                                                                                                                                                         | 11 (эксп.)                     | + вообще нет GC (no-op), − память не очищает                |
| **~2023 /<br> JDK 21**              | **Generational ZGC (GenZGC)**                                                                                                                                                                                                                                       | 21+                            | + low pause + young/old efficiency; − новый, требует tuning |
| **~2023+ /<br> JDK 24+**            | **GenShen (Gen Shenandoah)**                                                                                                                                                                                                                                        | 24+                            | + concurrent gen compaction; − niche                        |

### 📌 Короткие смысловые тезисы по каждому
- **Serial GC** — максимально простой, подходит для малых приложений.    
- **Parallel GC** — ориент. на максимальную пропускную способность, стандарт до JDK 9. 
- **CMS** — минимальные паузы, но риск фрагментации, устарел.    
- **G1 GC** — default с Java 9, сбалансированный, региональный сбор.    
- **ZGC** — ультранизкие паузы (микро <10 мс), хорошо для больших heap.    
- **Shenandoah GC** — похожие цели как ZGC, другое исполнение.    
- **Epsilon GC** — экспериментальный «нулевой» GC (не освобождает память).    
- **Generational ZGC / GenShen** — современные улучшения ZGC/Shenandoah с разделением на поколения.

## 🧠 **Какой GC выбирать** (кратко)

| GC                          | Когда выбирать                               | Коротко почему                                    |
| --------------------------- | -------------------------------------------- | ------------------------------------------------- |
| **Serial** GC               | Маленькие сервисы, CLI, тесты                | Минимальный overhead, простой                     |
| **Parallel** GC             | Batch, расчёты, максимум throughput          | Быстро обрабатывает много данных, паузы допустимы |
| **CMS** (устарел)           | Не выбирать                                  | Заменён более современными GC                     |
| **G1** GC                   | 🟢 По умолчанию (микросервисы, обычный прод) | Баланс latency/throughput, предсказуемые паузы    |
| **ZGC**                     | Low-latency, heap 8GB+                       | Паузы <10ms, масштабируется                       |
| **Shenandoah**              | Low-latency, альтернатива ZGC                | Конкурентная компакция                            |
| **Epsilon** GC              | Тесты, short-lived процессы                  | Нет сборки мусора вообще                          |
| **Generational ZGC**        | Современный low-latency (Java 21+)           | Быстрее ZGC за счёт поколений                     |
| **Generational Shenandoah** | Специфические low-latency кейсы              | Улучшенный Shenandoah                             |

### 📌 Быстрое правило
- Обычный прод → **G1**    
- Очень важна низкая latency → **ZGC (Java 21+)**    
- Максимальный throughput → **Parallel**    
- Тест / benchmarking → **Epsilon**
