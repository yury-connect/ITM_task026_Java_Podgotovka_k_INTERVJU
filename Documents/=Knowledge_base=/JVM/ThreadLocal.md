# 🧵 **ThreadLocal** — простыми словами

---
## 🎯 Что такое ThreadLocal

**ThreadLocal** — это специальный класс в Java, который позволяет хранить **отдельную копию переменной для каждого потока**.

Каждый поток видит **своё собственное** значение переменной и не мешает другим потокам.

---
## 🧠 Простая аналогия

**ThreadLocal** — это как **личный шкафчик** у каждого сотрудника в раздевалке:
	
- У каждого потока (сотрудника) есть свой шкафчик (ThreadLocal-переменная).
    
- Он может положить туда что угодно (set), взять оттуда (get) и никто другой не видит его содержимое.
    
- Два разных сотрудника никогда не перепутают свои вещи (значения).

---
## 📝 Пример
```java
public class UserContext {
    // Один объект ThreadLocal для всего приложения
    private static ThreadLocal<String> currentUser = new ThreadLocal<>();

    public static void setUser(String user) {
        currentUser.set(user);  // кладём в личный шкафчик текущего потока
    }

    public static String getUser() {
        return currentUser.get();  // достаём из личного шкафчика текущего потока
    }

    public static void clear() {
        currentUser.remove();  // очищаем (важно, чтобы не было утечек!)
    }
}
```

```java
// Поток 1 (пользователь Alice)
UserContext.setUser("Alice");
String user = UserContext.getUser();  // "Alice"

// Поток 2 (пользователь Bob) — одновременно
UserContext.setUser("Bob");
String user = UserContext.getUser();  // "Bob"

// Значения НЕ ПЕРЕМЕШИВАЮТСЯ!
// Поток 1 видит "Alice", поток 2 видит "Bob"
```

## 🔧 Где используется ThreadLocal

|Где|Что хранит|
|---|---|
|**Spring Security**|Информацию о текущем пользователе (`SecurityContextHolder`)|
|**Spring Transaction**|Статус текущей транзакции (`TransactionSynchronizationManager`)|
|**RequestContextHolder**|Данные текущего HTTP-запроса|
|**SimpleDateFormat**|Безопасное форматирование дат без создания нового объекта|
|**Сборщики логов (MDC)**|`userId`, `requestId` для сквозного логирования|

---
## ⚠️ Важное предупреждение

### 1️⃣ **Очищайте ThreadLocal после использования**
```java
try {
    UserContext.setUser("Alice");
    // ... делаем работу ...
} finally {
    UserContext.clear();  // ✅ ВАЖНО! Иначе утечка памяти в пуле потоков
}
```
**Почему:** В веб-приложениях потоки переиспользуются (пул потоков). Если не очистить, следующий запрос в том же потоке увидит старые данные.

### 2️⃣ **Не используйте для передачи данных между потоками**
```java
// ❌ Так НЕ РАБОТАЕТ!
new Thread(() -> {
    String user = UserContext.getUser();  // null! Это другой поток
}).start();
```

## 📊 Сравнение

|Концепция|Видимость|Где|
|---|---|---|
|**Обычная переменная**|Один поток|Внутри метода|
|**Static переменная**|Все потоки|Весь JVM|
|**ThreadLocal**|**Только текущий поток**|Весь код текущего потока|

---
## ✅ Итог одной фразой

> **ThreadLocal** — это личная переменная для каждого потока.  
> Поток А не видит, что положил поток Б. Идеально для хранения данных текущего пользователя, запроса или транзакции.

**Запомни:** Если не очистить ThreadLocal → будет утечка памяти! Всегда используй `remove()` в `finally`.
