## 🏆 ТОП-5 самых востребованных технологий API

### 1. REST
- **Сфера**: Публичные API, веб-сервисы, микросервисы    
- **Формат**: JSON / XML    
- **Протокол**: HTTP/1.1, HTTP/2    
- **Плюсы**: Простота, человекочитаемость, широкая поддержка    
- **Минусы**: Много данных (over-fetching), медленнее бинарных протоколов    
- **Когда использовать**: Стандартное веб-API, публичные эндпоинты, CRUD    

---
### 2. gRPC
- **Сфера**: Микросервисы, высоконагруженные системы, межсервисное общение    
- **Формат**: Protobuf (бинарный)    
- **Протокол**: HTTP/2    
- **Плюсы**: Высокая скорость, строгий контракт, поддержка streaming    
- **Минусы**: Нечитаемый бинарный формат, сложнее отладка, ограниченная поддержка браузеров    
- **Когда использовать**: Внутренние сервисы, high-load, реальное время    

---
### 3. GraphQL
- **Сфера**: Сложные клиентские приложения (мобилки, SPA), агрегация данных    
- **Формат**: JSON    
- **Протокол**: HTTP    
- **Плюсы**: Клиент сам выбирает поля, один запрос вместо нескольких    
- **Минусы**: Сложнее кэшировать, риск сложных запросов (N+1 проблема)    
- **Когда использовать**: Клиенты с разными потребностями в данных, дашборды    

---
### 4. WebSocket
- **Сфера**: Реальное время (чаты, игры, трейдинг, нотификации)    
- **Формат**: Любой (JSON, бинарный)    
- **Протокол**: WebSocket (поверх TCP)    
- **Плюсы**: Полнодуплексная связь, постоянное соединение, низкая задержка    
- **Минусы**: Сложнее масштабировать, требует поддержки соединений    
- **Когда использовать**: Приложения реального времени, биржевые данные    

---
### 5. SOAP
- **Сфера**: Корпоративные системы, банкинг, страхование, legacy-системы    
- **Формат**: XML    
- **Протокол**: HTTP, SMTP, JMS и другие    
- **Плюсы**: Строгая контрактность, безопасность (WS-Security), транзакционность    
- **Минусы**: Громоздкий, медленный, сложный    
- **Когда использовать**: Enterprise-интеграции, где нужна строгая гарантия и безопасность
